# Спецификация: Домен рынка скота

## Результат
Обеспечить размещение, поиск и управление объявлениями о сельскохозяйственном скоте (крупный рогатый скот, лошади, овцы, козы, свиньи, птица и т.д.). Поддержать операции продажи, разведения, аренды и выставки. Обеспечить соответствие российским правилам торговли скотом и требованиям идентификации животных.

## Область и границы
**Включает:**
- Объявления о скоте с атрибутами, специфичными для вида (бирка для ушей, паспорт, свидетельство о рождении, результаты тестов на здоровье, показатели продуктивности)
- Гео-поиск: поиск по радиусу (1-100 км) от местоположения пользователя
- Фильтрация: по виду, породе, возрасту, полу, цене, местоположению, типу объявления (продажа, разведение, выставка, усыновление, stud_service)
- Рабочий процесс модерации: пре-модерация (объявление появляется только после одобрения модератором)
- Действия пользователей: создание, редактирование, удаление собственных объявлений, избранные, поделиться
- Интеграция с доменом животного (связывание объявления с сущностью животного)
- Интеграция с доменом идентификации (информация о продавце/владельце)
- Соответствие российским законам о торговле скотом, ветеринарным правилам и 152-ФЗ для персональных данных

**Исключает:**
- Системы аукционов/торгов - отложено
- Обработка платежей (внутриприложение транзакции) - отложено на этап 2
- Логистика доставки/отправки для скота - отложено (специализированный транспорт)
- Оценки/отзывы пользователей для продавцов - отложено
- Отслеживание перемещения скота/журналы транспортировки - отложено на этап 2
- Интеграция с государственными системами вроде "Merкурий" - отложено

## Ограничения
- **Юридическое:** Должно соответствовать Федеральному закону РФ "О ветеринарной медицине", нормам идентификации скота и 152-ФЗ для обработки персональных данных.
- **Целостность данных:** Предотвращение дублирования записей о скоте (уникальность бирки для ушей/паспорта).
- **Расширяемость:** Поддержка атрибутов, специфичных для вида, через JSONB или расширяющие сущности.
- **Производительность:** Поиск скота по бирке для ушей/паспорту < 500мс; гео-поиск < 2с.
- **Масштабируемость:** Поддержка 50k+ активных объявлений о скоте (меньший объем, чем у домашних животных, но более высокая стоимость).
- **Технологии:** Должно соответствовать выбранному стеку (NestJS, TypeScript, PostgreSQL).
- **Удобство использования:** Интерфейс должен вмещать сложные атрибуты скота (бирки для ушей, сертификаты здоровья).

## Предыдущие решения
- Скот обрабатывается аналогично домашним животным в домене животного, но с разными требованиями к атрибутам.
- Справочные данные по видам и породам совместно используются между рынком домашних животных и рынком скота (управляются через домен администрирования).
- Атрибуты скота существенно различаются по виду (например, крупный рогатый скот требует бирок для ушей, а птица требует другого отслеживания).
- Российские правила требуют бирок для ушей/паспортирования для большинства видов скота.
- Мы храним минимальные персональные данные владельца в объявлениях (только ссылка на userId) для соответствия 152-ФЗ; полные данные владельца находятся в домене идентификации.
- Структура сущности объявления совместно используется между рынком домашних животных и рынком скота, но с разными правилами валидации и наборами атрибутов.

## Трассируемость НФТ
Эта спецификация охватывает следующие нефункциональные требования:
- **Производительность (НФТ-ПРОИЗВ)**: Поиск скота по бирке для ушей/паспорту < 500мс; гео-поиск < 2с (см. docs/02-requirements/nfr/performance.md)
- **Безопасность (НФТ-БЕЗОП)**: Объявления не раскрывают персональные данные сверх разрешенного (телефон/социальные сети после модерации); соблюдение российских законов о торговле скотом и требований идентификации животных (см. docs/02-requirements/nfr/security.md)
- **Доступность (НФТ-ДОСТУП)**: Интерфейс рынка скота следует рекомендациям WCAG 2.1 AA (см. docs/02-requirements/nfr/accessibility.md)

## User Stories

### Управление рынком скота
**UC-LM-01:** Как фермер или владелец скота, я хочу легко создавать и управлять своими объявлениями о скоте, чтобы я мог эффективно продавать, разводить или демонстрировать своих животных.
- Критерии приёмки:
  - Форма создания объявления загружается за <2с
  - Выбор вида и породы использует выпадающие списки с поиском и опциями, специфичными для скота
  - Динамическая форма подстраивает поля в зависимости от выбранного вида (бирки для ушей, данные паспорта, сертификаты здоровья и т.д.)
  - Чёткая визуальная обратная связь о валидности/невалидности формы
  - Подтверждение успеха с идентификатором объявления показывается сразу
  - Возможность пропустить необязательные поля и вернуться к ним позже
  - Выбор местоположения на карте с поиском по радиусу
  - Загрузка фотографий с предпросмотром и оптимизацией

**UC-LM-02:** Как покупатель или заводчик скота, я хочу легко находить скот, соответствующий моим критериям, чтобы я мог эффективно находить подходящих животных для покупки, разведения или выставки.
- Критерии приёмки:
  - Атрибуты скота видны в карточках объявлений (вид, порода, возраст, пол, фотографии, цена, местоположение)
  - Опции фильтрации заметны и легко применяются (вид, порода, возраст, пол, цена, местоположение, тип объявления)
  - Поиск на карте с селектором радиуса (1-100 км)
  - Результаты поиска загружаются быстро (<1с) с бесконечной прокруткой
  - Чёткое разграничение поисковых атрибутов и приватных данных
  - Сохранённые поиски для часто используемых критериев
  - Опции сортировки (новизна, цена, расстояние)
  - Раскрытие прямых контактов после модерации (телефон, ссылки Telegram/VK)

### Модерация и обеспечение качества
**UC-LM-03:** Как модератор, я хочу эффективно проверять объявления о скоте, чтобы я мог обеспечить качество, соответствие требованиям и безопасность на рынке.
- Критерии приёмки:
  - Очередь модерации загружается за <2с с чёткими индикаторами статуса
  - Детали объявления показывают все необходимые атрибуты скота для проверки (бирка для ушей, паспорт, сертификаты здоровья)
  - Одобрение/отклонение в один клик с необязательными комментариями
  - Опции массовой модерации для похожих объявлений
  - Чёткий чек-лист соответствия российским правилам торговли скотом
  - Система уведомлений для пользователей при изменении статуса объявления
  - Журнал аудита действий модерации для подотчётности
  - Возможность запросить дополнительную информацию у продавца

**UC-LM-04:** Как пользователь, заботящийся о приватности и контроле, я хочу управлять видимостью и статусом владения своими объявлениями о скоте, чтобы я мог защитить свою информацию и соблюдать личные предпочтения.
- Критерии приёмки:
  - Чёткий переключатель между активным/неактивным состоянием объявления
  - Визуальная индикация, когда объявление неактивно (затемнено, метка)
  - Простой процесс повторной активации с подтверждением
  - Чёткое объяснение того, что происходит с данными объявления при деактивации
  - Возможность легко добавлять/удалять информацию об идентификации скота
  - Контроль над тем, какая персональная информация видна после модерации (телефон, ссылки на социальные сети)
  - Лёгкое удаление завершённых/просроченных объявлений

**UC-LM-05:** Как торговец скотом или фермер, я хочу отслеживать важную информацию о здоровье и продуктивности, чтобы я мог принимать обоснованные решения об уходе за скотом и его торговом потенциале.
- Критерии приёмки:
  - Структурированный ввод вакцинаций, тестов, лечения
  - Отслеживание показателей продуктивности (надой молока, прирост веса, количество потомства)
  - Сводка о здоровье видна с первого взгляда в объявлении
  - Интеграция с медицинскими записями животного из домена животного
  - Возможность экспорта записей о здоровье и продуктивности (будущее улучшение)
  - Напоминания о предстоящих проверках здоровья или лечении (будущее улучшение)

## Разбивка задач
1. **Бэкенд (NestJS)**
   - [ ] Создать модуль `livestock-marketplace`
   - [ ] Определить сущность Объявления (общая структура с pet-marketplace, но с разной валидацией)
   - [ ] Реализовать правила валидации, специфичные для вида (например, если вид=крупный рогатый скот, earTagId обязателен; если вид=птица, разные правила)
   - [ ] Создать ListingController (CRUD, поиск, действия модерации)
   - [ ] Создать ListingService (логика бизнеса: валидация, гео-поиск, рабочий процесс модерации)
   - [ ] Настроить индексы базы данных: местоположение (гео), статус, тип объявления, speciesId (через животное), sellerId
   - [ ] Реализовать конечные точки модерации (для роли модератора)
   - [ ] Написать модульные и интеграционные тесты для жизненного цикла объявления и поиска
   - [ ] Создать документацию OpenAPI для конечных точек объявления

2. **Фронтенд (React)**
   - [ ] Создать страницы объявлений: Создать объявление, Редактировать объявление, Список объявлений (поиск/просмотр), Детали объявления
   - [ ] Реализовать динамическую форму, которая изменяет поля в зависимости от выбранного вида/породы (специфично для скота)
   - [ ] Реализовать выбор местоположения на карте (используя API Яндекс.Карт)
   - [ ] Создать компонент карточки объявления для видового/спискового представления
   - [ ] Реализовать функционал избранного и «поделиться»
   - [ ] Интегрировать с доменом идентификации для отображения информации о продавце (ограниченная тем, что разрешено после модерации)
   - [ ] Написать модульные и end-to-end тесты для потоков работы с объявлением

3. **Инфраструктура**
   - [ ] Активировать расширение PostGIS в PostgreSQL (или использовать альтернативную гео-индексацию)
   - [ ] Настроить схему Prisma для Объявления и связанных сущностей (общая с pet-marketplace)
   - [ ] Настроить сегмент облачного хранилища для медиа объявлений
   - [ ] Реализовать загрузку изображений через предварительно подписанные URL (фронтенд -> хранилище)
   - [ ] Добавить кэширующий слой для частых поисковых запросов (Redis)
   - [ ] Реализовать ограничение частоты запросов для конечных точек создания объявлений

## Критерии верификации
- [ ] Модульные тесты >90% покрытия для модуля livestock-marketplace (backend)
- [ ] Интеграционные тесты покрывают: создание объявления (валидное/невалидное в зависимости от вида), поиск по гео-радиусу, фильтрация, рабочий процесс модерации, переходы статусов
- [ ] End-to-end тесты покрывают: пользователь создает объявление, ищет объявление, просматривает детали объявления, редактирует объявление
- [ ] Ручное тестирование: проверка точности гео-поиска, валидация, зависящая от вида, рабочий процесс модерации, загрузка изображений
- [ ] Производительность: гео-поиск с 50k объявлениями возвращает результат за <1с для 95% запросов
- [ ] Соответствие: объявления не раскрывают персональные данные сверх разрешенного (телефон/социальные сети после модерации); соблюдение российских законов о торговле скотом и требований идентификации животных
- [ ] Документация: спецификация OpenAPI сгенерирована и доступна
- [ ] Трассируемость НФТ: проверить, что требования производительности, безопасности и доступности корректно учтены и документированы
