# Спецификация: Pet vs Livestock — расходящиеся правила (хард-сплит)

## Результат
[ADR-0002](../04-decisions/0002-hard-split-markets.md) предписывает жёсткое разделение рынков **pet** и
**livestock**, но per-market спеки были почти идентичны. Этот документ — **нормативная таблица конкретных
различий**, которые обязан реализовать бэкенд; без неё сплит — лишь UI-роутинг.

Рынок объявления выводится из **вида** животного (`species.market` ∈ {`pet`, `livestock`} — атрибут справочных
данных админки). Все правила ниже применяются по рынку животного.

## Расходящиеся правила (нормативно)

| Правило | Pet | Livestock |
|---|---|---|
| **Допустимые `listing_type`** | `sale`, `adoption`, `breeding`, `stud_service`, `show` | `sale`, `breeding`, `stud_service` (без `adoption`/`show`) |
| **`quantity`** | всегда `1` (питомец — особь) | `>= 1` (голова или партия одного вида) |
| **Требование цены** | обязательна для `sale`; NULL/0 для `adoption` | обязательна для `sale` и `stud_service` |
| **Акцент идентификации** | `microchip_id` (уникален), прививки в `health_records` | `tattoo_brand_id`/бирка/паспорт (уникальны), данные стада в `health_records` |
| **Лимит раскрытия контактов** | 10 / час / пользователь | 5 / час / пользователь (анти-скрейпинг) |
| **Чек-лист модерации** | благополучие питомцев, список запрещённых видов, подлинность фото | вет/сан-нормы, ветеринарные атрибуты, консистентность партии |
| **Пригодность к разведению** | тот же предикат (см. `05-matching-domain.md`) по pet-видам | тот же предикат по livestock-видам |
| **Гео-поиск** | идентичный механизм (`lat/lng` + Haversine) | идентичный |

## Инвариант сплита
Объявление **не должно** смешивать рынки: рынок вида животного определяет рынок объявления, и правила
`listing_type` / `quantity` / цены выше валидируются соответственно в сервис-слое. Pet-животное не может породить
livestock-объявление и наоборот (ADR-0002).

## Замечание по реализации
- Добавить `species.market VARCHAR(10) CHECK (market IN ('pet','livestock'))` в справочные данные (seed по видам).
- Валидировать `listing_type`, `quantity` и правила цены в сервисе листинга по рынку животного.
- Enum БД (`listing_type`) остаётся общим; **допустимое подмножество** по рынку — правило сервис-слоя из этой таблицы.
