# Спецификации ZooLink

Этот каталог содержит артефакты Spec-Driven Development (SDD) для проекта ZooLink.
Каждая спецификация следует шаблону SDD и служит источником истины для разработки.

## Спецификации

1. [01-identity-domain.md](01-identity-domain.md) - Аутентификация, авторизация и управление пользователями
2. [02-animal-domain.md](02-animal-domain.md) - Сущность животного как агрегатный корень
3. [03-pet-marketplace-domain.md](03-pet-marketplace-domain.md) - Маркетплейс объявлений о домашних животных
4. [04-livestock-marketplace-domain.md](04-livestock-marketplace-domain.md) - Маркетплейс объявлений о сельскохозяйственном скоте
5. [05-matching-domain.md](05-matching-domain.md) - Функционал подбора для разведения
6. [06-admin-domain.md](06-admin-domain.md) - Административные и модерационные функции
7. [07-geo-search-service.md](07-geo-search-service.md) - Возможности гео-поиска
8. [08-frontend-architecture.md](08-frontend-architecture.md) - Трехслойная архитектура фронтенда (View→Domain→System)

10. [10-implementation-roadmap.md](10-implementation-roadmap.md) - Поэтапный план внедрения

## Как использовать эти спецификации

1. **Прочтите спецификацию** для функции, над которой вы работаете
2. **Проверьте реализацию** по разбивке задач и критериям верификации спецификации
3. **Обновите спецификацию** если требования изменятся (спецификации являются источником истины)
4. **Создавайте тесты** на основе критериев верификации
5. **Внедряйте функции** следуя разбивке задач

## Процесс S-D-D (Spec-Driven Development)

1. **Напишите или обновите спецификацию** перед началом работы
2. **Проверьте спецификацию** с командой/заинтересованными сторонами
3. **Внедрите** в соответствии со спецификацией
4. **Проверьте** реализацию соответствует критериям спецификации
5. **Обновите спецификацию** если узнали что-то новое во время внедрения

Все спецификации написаны в формате Markdown и должны поддерживаться в актуальном состоянии как единый источник истины.
