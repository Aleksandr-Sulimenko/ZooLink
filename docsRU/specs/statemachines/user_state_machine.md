# Спецификация конечного автомата состояний пользователя

## Обзор
Определяет состояния жизненного цикла и переходы для учетной записи пользователя в системе ZooLink.

## Состояния

| Состояние | Описание | Действия при входе | Действия при выходе |
|-----------|----------|-------------------|---------------------|
| **НЕПРОВЕРЕННЫЙ** | Исходное состояние после регистрации; пользователь не может выполнять основные действия (создание объявлений и т.д.) | - Отправить код верификации через SMS/OAuth<br>- Зарегистрировать попытку регистрации | - Очистить код верификации из памяти (если отправлен) |
| **ОЖИДАНИЕ_ВЕРИФИКАЦИИ** | Процесс верификации запущен; ожидает ответа пользователя | - Запустить таймер попыток верификации<br>- Увеличить счетчик попыток верификации | - Остановить таймер верификации |
| **ПРОВЕРЕННЫЙ** | Пользователь успешно проверил свою идентичность через SMS/OAuth | - Предоставить базовый доступ к платформе<br>- Зарегистрировать успешную верификацию | - Нет |
| **АКТИВНЫЙ** | Полностью проверенный пользователь с полным доступом к функциям платформы | - Включить создание/модификацию объявлений<br>- Предоставить доступ ко всем основным функциям | - Нет |
| **ПРИОСТАНОВЛЕН** | Временно ограниченный доступ из-за нарушений политики | - Отключить создание/модификацию объявлений<br>- Отправить уведомление о приостановке<br>- Зарегистрировать причину приостановки | - Нет |
| **ДЕАКТИВИРОВАН** | Постоянно деактивированный аккаунт (по запросу пользователя или действие админа) | - Анонимизировать персональные данные согласно GDPR<br>- Отозвать все токены доступа<br>- Зарегистрировать деактивацию | - Нет |

## Переходы между состояниями

| Из состояния | В состояние | Триггер | Условие сохранности | Действие |
|--------------|-------------|---------|---------------------|----------|
| НЕПРОВЕРЕННЫЙ | ОЖИДАНИЕ_ВЕРИФИКАЦИИ | Регистрация завершена | Выбран метод верификации (SMS/OAuth) | Отправить код верификации |
| ОЖИДАНИЕ_ВЕРИФИКАЦИИ | ПРОВЕРЕННЫЙ | Код верификации отправлен | Код совпадает && попыток < MAX_ATTEMPTS | Очистить данные верификации |
| ОЖИДАНИЕ_ВЕРИФИКАЦИИ | НЕПРОВЕРЕННЫЙ | Верификация провалена | попыток >= MAX_ATTEMPTS || пользователь запросил повторную отправку | Увеличить счетчик попыток; опционально переслать код |
| ОЖИДАНИЕ_ВЕРИФИКАЦИИ | НЕПРОВЕРЕННЫЙ | Регистрация брошена | Таймаут сессии || пользователь перешел на другую страницу | Очистить временные данные |
| ПРОВЕРЕННЫЙ | АКТИВНЫЙ | Завершение профиля (если требуется) | Все обязательные поля профиля заполнены | Активировать полный аккаунт |
| ПРОВЕРЕННЫЙ | АКТИВНЫЙ | Автоматическая активация (без требований к профилю) | Время elapsed > VERIFICATION_GRACE_PERIOD | Активировать полный аккаунт |
| АКТИВНЫЙ | ПРИОСТАНОВЛЕН | Действие модерации | Тяжесть нарушения >= ПОРОГ_ПРИОСТАНОВЛЕНИЯ | Уведомить пользователя; зарегистрировать нарушение |
| ПРИОСТАНОВЛЕН | АКТИВНЫЙ | Апелляция успешна | Результат пересмотра модерации = ОДОБРЕНО | Восстановить доступ; зарегистрировать результат апелляции |
| ПРИОСТАНОВЛЕН | ДЕАКТИВИРОВАН | Апелляция провалена ИЛИ приостановка истекла | Результат пересмотра модерации = ОТКЛОНЕНО || время в приостановке > MAX_SUSPENSION_DURATION | Инициировать процесс деактивации |
| АКТИВНЫЙ | ДЕАКТИВИРОВАН | Запрос пользователя | Инициировано удаление аккаунта пользователем | Анонимизировать данные; отозвать токены |
| АКТИВНЫЙ | ДЕАКТИВИРОВАН | Действие администратора | Тяжесть нарушения = ПОРОГ_ТЕРМИНАЦИИ | Анонимизировать данные; зарегистрировать действие админа |
| * | ДЕАКТИВИРОВАН | Системный мандат | Требование законодательства (например, право на забвение) | Анонимизировать данные; зарегистрировать действие соответствия |

## Константы и конфигурация
- `MAX_ATTEMPTS`: 5 (попытки ввода кода верификации)
- `VERIFICATION_GRACE_PERIOD`: 24 часа (время на завершение верификации)
- `MAX_SUSPENSION_DURATION`: 30 дней (максимальная продолжительность приостановки перед авто-деактивацией)
- `SUSPENSION_THRESHOLD`: Балл нарушения >= 70 ( basado на весах типов нарушений)
- `TERMINATION_THRESHOLD`: Балл нарушения >= 90

## Замечания
- Все переходы между состояниями логируются с временной отметкой, ID пользователя и контекстом триггера для журналов аудита.
- Терминальные состояния: ПРОВЕРЕННЫЙ, АКТИВНЫЙ, ПРИОСТАНОВЛЕН, ДЕАКТИВИРОВАН (НЕПРОВЕРЕННЫЙ и ОЖИДАНИЕ_ВЕРИФИКАЦИИ являются переходными).
- Из состояния ДЕАКТИВИРОВАН отсутствуют возможные переходы (аккаунт постоянно удален из активных систем).
