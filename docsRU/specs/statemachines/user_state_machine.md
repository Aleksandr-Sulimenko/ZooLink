# Спецификация автомата состояний пользователя

## Обзор
Определяет состояния lifecycle и переходы для учетной записи пользователя в системе ZooLink.

## Состояния

| Состояние | Описание | Действия при входе | Действия при выходе |
|-----------|----------|-------------------|---------------------|
| **UNVERIFIED** | Начальное состояние после регистрации; пользователь не может выполнять основные действия (создание листингов и т.д.) | - Отправить код верификации через SMS/OAuth<br>- Запомнить попытку регистрации | - Очистить код верификации из памяти (если отправлен) |
| **PENDING_VERIFICATION** | Процесс верификации инициирован; ожидание ответа от пользователя | - Запустить таймер попытки верификации<br>- Увеличить счетчик попыток верификации | - Остановить таймер верификации |
| **VERIFIED** | Пользователь успешно прошел проверку идентичности через SMS/OAuth | - Предоставить базовый доступ к платформе<br>- Логировать успешную верификацию | - Нет |
| **ACTIVE** | Полностью проверенный пользователь с полным доступом к функциям платформы | - Включить создание/модификацию листингов<br>- Предоставить доступ ко всем основным функциям | - Нет |
| **SUSPENDED** | Временно ограниченный доступ из-за нарушений политики | - Отключить создание/модификацию листингов<br>- Отправить уведомление о приостановлении<br>- Журналировать причину приостановления | - Нет |
| **DEACTIVATED** | Постоянно деактивированный аккаунт (по запросу пользователя или действию администратора) | - Анонимизировать персональные данные в соответствии с GDPR<br>- Отозвать все токены доступа<br>- Логировать деактивацию | - Нет |

## Переходы состояний

| От состояния | К состоянию | Триггер | Условие срабатывания (Guard) | Действие |
|--------------|-------------|---------|------------------------------|----------|
| UNVERIFIED | PENDING_VERIFICATION | Регистрация завершена | Выбран метод верификации (SMS/OAuth) | Отправить код верификации |
| PENDING_VERIFICATION | VERIFIED | Код верификации подан | Код совпадает && попытки < MAX_ATTEMPTS | Очистить данные верификации |
| PENDING_VERIFICATION | UNVERIFIED | Верификация провалилась | попытки >= MAX_ATTEMPTS || пользователь запросил повторную отправку | Увеличить счетчик попыток; опционально переслать код |
| PENDING_VERIFICATION | UNVERIFIED | Регистрация оставлена | Таймаут сессии || пользователь ушел с экрана | Очистить временные данные |
| VERIFIED | ACTIVE | Завершение профиля (если требуется) | Все обязательные поля профиля заполнены | Активировать полный аккаунт |
| VERIFIED | ACTIVE | Автоматическая активация (без требования профиля) | Прошло время > VERIFICATION_GRACE_PERIOD | Активировать полный аккаунт |
| ACTIVE | SUSPENDED | Действие модерации | Серьезность нарушения >= ПОРОГ ПРИОСТАНОВЛЕНИЯ | Уведомить пользователя; залогировать нарушение |
| SUSPENDED | ACTIVE | Апелляция успешная | Результат рассмотрения апелляции = ОДОБРЕНО | Восстановить доступ; залогировать результат апелляции |
| SUSPENDED | DEACTIVATED | Апелляция провалилась ИЛИ приостановление истекло | Результат рассмотрения апелляции = ОТКЛОНЕНО || время в приостановлении > MAX_SUSPENSION_DURATION | Инициировать процесс деактивации |
| ACTIVE | DEACTIVATED | Запрос пользователя | Инициировано пользователем удаление аккаунта | Анонимизировать данные; отозвать токены |
| ACTIVE | DEACTIVATED | Административное действие | Серьезность нарушения = ПОРОГ ТЕРМИНАЦИИ | Анонимизировать данные; залогировать действие администратора |
| * | DEACTIVATED | Системный мандат | Юридическое требование (например, право быть забытым) | Анонимизировать данные; залогировать действие соответствия |

## Константы и конфигурация
- `MAX_ATTEMPTS`: 5 (попыток верификации)
- `VERIFICATION_GRACE_PERIOD`: 24 часа (время на завершение верификации)
- `MAX_SUSPENSION_DURATION`: 30 дней (максимальная продолжительность приостановления перед авто-деактивацией)
- `SUSPENSION_THRESHOLD`: Оценка нарушения >= 70 (на основе весов типов нарушений)
- `TERMINATION_THRESHOLD`: Оценка нарушения >= 90

## Примечания
- Все переходы состояний логируются с отметкой времени, идентификатором пользователя и контекстом триггера для аудита.
- Терминальные состояния: VERIFIED, ACTIVE, SUSPENDED, DEACTIVATED (UNVERIFIED и PENDING_VERIFICATION являются переходными).
- Из состояния DEACTIVATED переходов невозможен (аккаунт окончательно удален из активных систем).
