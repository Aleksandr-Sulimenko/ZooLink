# Спецификация конечного автомата состояний объявления

## Обзор
Определяет состояния жизненного цикла и переходы для объявления (животное на продажу/удочерение) в системе ZooLink.

## Состояния

| Состояние | Описание | Действия при входе | Действия при выходе |
|-----------|----------|-------------------|---------------------|
| **ЧЕРНОВИК** | Исходное состояние после создания объявления; видно только владельцу; недоступно в публичном поиске | - Назначить временный ID объявления<br>- Установить временной отметкой создания<br>- Проверить минимально требуемые поля (заголовок, цена, местоположение, ID животного) | - Очистить временные данные черновика |
| **ОЖИДАНИЕ_МОДЕРАЦИИ** | Объявление отправлено на рассмотрение; не видно в публичном поиске; ожидает действий модератора | - Увеличить счетчик очереди модерации<br>- Уведомить команду модерации<br>- Запустить таймер SLA модерации | - Остановить таймер SLA при быстром выходе |
| **АКТИВНО** | Объявление одобрено и видно в публичном поиске; доступно для покупки/удочерения | - Опубликовать в поисковых индексах<br>- Активировать видимость гео-поиска<br>- Установить временную метку публикации<br>- Активировать кнопки покупки/запроса | - Нет |
| **ИСТЕКШИЙ** | Объявление автоматически деактивировано после истечения срока; сохраняет историю | - Удалить из активных поисковых индексов<br>- Установить временную отметку истечения<br>- Уведомить владельца об истечении | - Нет |
| **ПРОДАНО** | Объявление отмечено как завершенное через успешную транзакцию; сохраняет историю | - Записать ID транзакции<br>- Установить временную отметку завершения<br>- Уведомить покупателя и продавца<br>- Запустить процесс трансфера собственности | - Нет |
| **ДЕАКТИВИРОВАНО** | Объявление вручную удалено владельцем или модератором; сохраняет историю | - Установить временную отметку деактивации<br>- Зафиксировать причину деактивации<br>- Уведомить заинтересованные стороны (если применимо) | - Нет |

## Переходы между состояниями

| Из состояния | В состояние | Триггер | Условие сохранности | Действие |
|--------------|-------------|---------|---------------------|----------|
| ЧЕРНОВИК | ОЖИДАНИЕ_МОДЕРАЦИИ | Владелец отправляет на рассмотрение | Все обязательные поля валидны && медиа загружены && цена >= MIN_LISTING_PRICE | Увеличить счетчик отправок |
| ЧЕРНОВИК | ЧЕРНОВИК | Владелец редактирует объявление | Пользователь является владельцем && объявление не истекло/не продано | Обновить поля; сбросить валидацию |
| ЧЕРНОВИК | ДЕАКТИВИРОВАНО | Владелец бросает черновик | Пользователь явно удаляет || automat. очистка по таймауту ЧЕРНОВИКА | Журналировать бросание; очистить временные данные |
| ОЖИДАНИЕ_МОДЕРАЦИИ | АКТИВНО | Модератор одобряет | Решение модерации = ОДОБРЕНО && нет нарушений политики | Опубликовать объявление; уведомить владельца |
| ОЖИДАНИЕ_МОДЕРАЦИИ | ДЕАКТИВИРОВАНО | Модератор отклоняет | Решение модерации = ОТКЛОНЕНО || нарушение политики найдено | Уведомить владельца с причиной; журнал отклонения |
| ОЖИДАНИЕ_МОДЕРАЦИИ | ИСТЕКШИЙ | Таймаут модерации истек | Нет действий модератора в течение MODERATION_SLA_HOURS | Автоматически отклонить; уведомить владельца |
| АКТИВНО | ИСТЕКШИЙ | Истек срок объявления | Время с публикации > LISTING_DURATION_DAYS && не продано | Удалить из поиска; уведомить владельца |
| АКТИВНО | ПРОДАНО | Транзакция завершена | Статус платежа = ПОДТВЕРЖДЕНО && покупатель подтвердил получение | Записать продажа; инициировать трансфер собственности |
| АКТИВНО | ДЕАКТИВИРОВАНО | Владелец снимает объявление | Пользователь является владельцем && объявление активно && не в транзакции | Уведомить заинтересованные стороны; журнал снятия |
| АКТИВНО | ДЕАКТИВИРОВАНО | Модератор удаляет | Решение модерации = УДАЛИТЬ_АКТИВНО || тяжелое нарушение политики | Уведомить владельца; журнал действия модерации |
| ПРОДАНО | ДЕАКТИВИРОВАНО | Очистка после продажи | Транзакция полностью завершена && собственность передана | Архивировать данные объявления; сохранить для истории |
| ИСТЕКШИЙ | ДЕАКТИВИРОВАНО | Владелец продлевает или удаляет | Инициировано продление удаление пользователем | Если продление: сброс в ЧЕРНОВИК; если удаление: архивировать |
| * | ДЕАКТИВИРОВАНО | Системный мандат | Требование законодательства (например, запрещенное животное) | Анонимизировать конфиденциальные данные; журнал соответствия |

## Константы и конфигурация
- `MIN_LISTING_PRICE`: 0 (бесплатные объявления разрешены) или 1 (минимальная единица валюты) - настраивается по региону
- `DRAFT_TIMEOUT`: 7 дней (автоочистка заброшенных черновиков)
- `MODERATION_SLA_HOURS`: 24 часа (окно проверки модерации)
- `LISTING_DURATION_DAYS`: 30 дней (стандартная продолжительность объявления; настраивается по типу объявления)
- `MAX_MEDIA_ITEMS`: 10 (максимум фото/видео на объявление)
- `MIN_TITLE_LENGTH`: 3 символа
- `MAX_TITLE_LENGTH`: 100 символов

## Замечания
- Все переходы между состояниями логируются с временной отметкой, ID объявления, ID пользователя (владельца/модератора) и контекстом триггера.
- Терминальные состояния: АКТИВНО, ИСТЕКШИЙ, ПРОДАНО, ДЕАКТИВИРОВАНО (ЧЕРНОВИК и ОЖИДАНИЕ_МОДЕРАЦИИ являются переходными).
- Из состояния ДЕАКТИВИРОВАНО допускаются переходы только: в ДЕАКТИВИРОВАНО (самопетля для обновлений) или системное мандатное архивирование.
- ИСТЕКШИЙ объявления могут быть продлены владельцем, эффективно сбрасывая в состояние ЧЕРНОВИК с существующими данными.
- Состояние ПРОДАНО указывает на успешное завершение транзакции, но не автоматически передаёт собственность - это отдельный процесс, запускаемый продажей.
