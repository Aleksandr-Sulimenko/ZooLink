# Спецификация автомата состояний листинга

## Обзор
Определяет состояния lifecycle и переходы для листинга (животное на продажу/усыновление) в системе ZooLink.

## Состояния

| Состояние | Описание | Действия при входе | Действия при выходе |
|-----------|----------|-------------------|---------------------|
| **DRAFT** | Начальное состояние после создания листинга; видно только владельцу; не доступно в поиске | - Назначить временный ID листинга<br>- Установить отметку времени создания<br>- Проверить минимально обязательные поля (название, цена, местоположение, ID животного) | - Очистить временные данные черновика |
| **PENDING_MODERATION** | Листинг отправлен на проверку; не виден в публичном поиске; ожидает действия модератора | - Увеличить счетчик очереди модерации<br>- Уведомить команду модерации<br>- Запустить таймер SLA модерации | - Остановить таймер SLA при быстром выходе |
| **ACTIVE** | Листинг одобрен и виден в публичном поиске; доступен для покупки/усыновления | - Опубликовать в поисковых индексах<br>- Активировать видимость геопоиска<br>- Установить отметку времени публикации<br>- Включить кнопки покупки/запроса | - Нет |
| **EXPIRED** | Листинг автоматически деактивирован после истечения срока; сохраняет историю | - Удалить из активных поисковых индексов<br>- Установить отметку времени истечения<br>- Уведомить владельца об истечении | - Нет |
| **SOLD** | Листинг помечен как завершенный через успешную транзакцию; сохраняет историю | - Записать ID транзакции<br>- Установить отметку времени завершения<br>- Уведомить покупателя и продавца<br>- Инициировать процесс передачи собственности | - Нет |
| **DEACTIVATED** | Листинг вручную удален владельцем или модератором; сохраняет историю | - Установить отметку времени деактивации<br>- Записать причину деактивации<br>- Уведомить заинтересованные стороны (если применимо) | - Нет |

## Переходы состояний

| От состояния | К состоянию | Триггер | Условие срабатывания (Guard) | Действие |
|--------------|-------------|---------|------------------------------|----------|
| DRAFT | PENDING_MODERATION | Владелец отправляет на проверку | Все обязательные поля валидны && медиа загружены && цена >= MIN_LISTING_PRICE | Увеличить счетчик отправок |
| DRAFT | DRAFT | Владелец редактирует листинг | Пользователь является владельцем && листинг не истек/не продан | Обновить поля; сбросить валидацию |
| DRAFT | DEACTIVATED | Владелец оставляет черновик | Пользователь явно удаляет || автоочистка после DRAFT_TIMEOUT | Залогировать оставление; очистить временные данные |
| PENDING_MODERATION | ACTIVE | Модератор одобряет | Решение модерации = ОДОБРЕНО && нет нарушений политики | Опубликовать листинг; уведомить владельца |
| PENDING_MODERATION | DEACTIVATED | Модератор отклоняет | Решение модерации = ОТКЛОНЕНО || найдено нарушение политики | Уведомить владельца с причиной; залогировать отклонение |
| PENDING_MODERATION | EXPIRED | Истек срок модерации | Нет действий модератора в течение MODERATION_SLA_HOURS | Автоматический отказ; уведомить владельца |
| ACTIVE | EXPIRED | Истек срок листинга | Время с публикации > LISTING_DURATION_DAYS && не продан | Удалить из поиска; уведомить владельца |
| ACTIVE | SOLD | Транзакция завершена | Статус оплаты = ПОДТВЕРЖДЕНО && покупатель подтвердил получение | Записать продажу; инициировать передачу собственности |
| ACTIVE | DEACTIVATED | Владелец снимает листинг | Пользователь является владельцем && листинг активен && не в транзакции | Уведомить заинтересованные стороны; залогировать снятие |
| ACTIVE | DEACTIVATED | Модератор удаляет | Решение модерации = УДАЛИТЬ_АКТИВНЫЙ || серьезное нарушение политики | Уведомить владельца; залогировать действие модератора |
| SOLD | DEACTIVATED | Очистка после продажи | Транзакция полностью завершена && собственность передана | Архивировать данные листинга; сохранить для истории |
| EXPIRED | DEACTIVATED | Владелец обновляет или удаляет | Пользователь инициализирует обновление ИЛИ явное удаление | Если обновление: сброс в DRAFT; если удаление: архивировать |
| * | DEACTIVATED | Системное требование | Юридическое требование (например, запрещенное животное) | Обезличить чувствительные данные; залогировать соблюдение |

## Константы и конфигурация
- `MIN_LISTING_PRICE`: 0 (бесплатные листинги разрешены) или 1 (минимальная единица валюты) - настраивается по региону
- `DRAFT_TIMEOUT`: 7 дней (автоочистка заброшенных черновиков)
- `MODERATION_SLA_HOURS`: 24 часа (окно проверки модерации)
- `LISTING_DURATION_DAYS`: 30 дней (стандартный срок листинга; настраивается по типу листинга)
- `MAX_MEDIA_ITEMS`: 10 (максимум фото/видео на листинг)
- `MIN_TITLE_LENGTH`: 3 символа
- `MAX_TITLE_LENGTH`: 100 символов

## Примечания
- Все переходы состояний логируются с отметкой времени, идентификатором листинга, идентификатором пользователя (владелец/модератор) и контекстом триггера.
- Терминальные состояния: ACTIVE, EXPIRED, SOLD, DEACTIVATED (DRAFT и PENDING_MODERATION являются переходными).
- Из DEACTIVATED переходов ограничены: только в себя (самопетля для обновлений) или системно-мandated архивация.
- EXPIRED листинги могут быть обновлены владельцем, фактически сбрасывая в состояние DRAFT с существующими данными.
- Состояние SOLD указывает на завершение транзакции, но не автоматически передает собственность - это отдельный процесс, запускаемый продажей.
