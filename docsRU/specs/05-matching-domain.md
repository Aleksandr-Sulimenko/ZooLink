# Спецификация: Домен подбора

## Результат
Включить специализированную логику подбора животных для размножения (спарринг) на основе различных критериев, таких как вид, родословная, состояние здоровья, местоположение и предпочтения владельцев. Облегчить взаимодействие между заинтересованными сторонами (для реализации через обмен контактами после модерации в MVP, чат отложен на этап 2).

## Область и границы
**Включает:**
- Профили животных, оптимизированные для подбора/поиска (отображение соответствующих черт: родословная, сертификаты здоровья, титулы и т.д.)
- Алгоритмы подбора: совместимость вида, медицинский скрининг, близость местоположения, предпочтения владельцев
- Интерфейс поиска/фильтрации для поиска потенциальных партнеров по спариванию
- Интеграция с доменом животного (данные о животном)
- Интеграция с доменом идентификации (информация о владельце, с элементами конфиденциальности)
- Интеграция с объявлениями: возможность преобразования профиля животного в объявление о разведении
- Пре-модерация для объявлений о разведении (тот же процесс, что и для других типов объявлений)
- Основной обмен контактами после модерации (телефон, ссылки на социальные сети)

**Исключает:**
- Генетический анализ/прогностическое разведение - отложено на этап 2
- Генерация и управление родословными картами - отложено
- Интеграция ветеринарных записей с ветеринарными системами - отложено
- Автоматизированные рекомендации по спариванию/ИИ-подбор - отложено
- Оплата за услуги разведения - отложено
- Система чата/обмена сообщениями - отложена на этап 2

## Ограничения
- **Юридическое:** Должно соответствовать правилам разведения животных в РФ и 152-ФЗ для персональных данных.
- **Качество данных:** Точность подбора зависит от полноты и проверенности профилей животных.
- **Производительность:** Поиск подбора должен возвращать результаты за <2с.
- **Масштабируемость:** Поддержка подбора по 50k+ животных, способных к размножению.
- **Технологии:** Должно соответствовать выбранному стеку (NestJS, TypeScript, PostgreSQL).
- **Удобство использования:** Интерфейс должен быть интуитивно понятен для заводчиков (может не быть техническими экспертами).

## Предыдущие решения
- Подбор изначально будет основываться на поисковых критериях, а не на сложных алгоритмах.
- Мы будем использовать ту же инфраструктуру объявлений/модерации для объявлений о разведении (тип объявления = "разведение").
- Домен животного будет хранить атрибуты, значимые для разведения: ID родословной, результаты тестов на здоровье, титулы и т.д.
- Конфиденциальность: информация о контакте владельца доступна только после того, как модератор одобрит объявление о разведении.
- Подбор на основе местоположения использует тот же гео-поиск, что и домены рынка.
- Мы различаем "объявления о разведении" (животные для спаривания) и "объявления о продаже" (потомство для продажи).

## Критерии верификации
- [ ] Модульные тесты >90% покрытия для функционала, связанного с подбором (backend)
- [ ] Интеграционные тесты покрывают: поиск для разведения (различные фильтры), отображение профиля животного, создание объявления о разведении
- [ ] End-to-end тесты покрывают: пользователь ищет партнера для разведения, просматривает профиль животного, создает объявление о разведении
- [ ] Ручное тестирование: проверка точности поиска, валидация, специфичная для разведения, рабочий процесс модерации
- [ ] Производительность: поиск для разведения с 50k животными возвращает результат за <1с для 95% запросов
- [ ] Соответствие: соблюдение российских правил разведения животных; персональные данные владельца защищены до момента после модерации
- [ ] Документация: спецификация OpenAPI сгенерирована и доступна
