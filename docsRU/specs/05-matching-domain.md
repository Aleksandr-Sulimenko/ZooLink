# Спецификация: Домен подбора

## Результат
Включить специализированную логику подбора животных для размножения (спаривание) на основе различных критериев, таких как вид, родословная, состояние здоровья, местоположение и предпочтения владельцев. Облегчить взаимодействие между заинтересованными сторонами (для реализации через обмен контактами после модерации в MVP, чат отложен на этап 2).

## Область и границы
**Включает:**
- Профили животных, оптимизированные для подбора/поиска (отображение соответствующих черт: родословная, сертификаты здоровья, титулы и т.д.)
- Алгоритмы подбора: совместимость вида, медицинский скрининг, близость местоположения, предпочтения владельцев
- Интерфейс поиска/фильтрации для поиска потенциальных партнеров по спариванию
- Интеграция с доменом животного (данные о животном)
- Интеграция с доменом идентификации (информация о владельце, с элементами конфиденциальности)
- Интеграция с объявлениями: возможность преобразования профиля животного в объявление о разведении
- Пре-модерация для объявлений о разведении (тот же процесс, что и для других типов объявлений)
- Основной обмен контактами после модерации (телефон, ссылки на социальные сети)

**Исключает:**
- Генетический анализ/прогностическое разведение - отложено на этап 2
- Генерация и управление родословными картами - отложено
- Интеграция ветеринарных записей с ветеринарными системами - отложено
- Автоматизированные рекомендации по спариванию/ИИ-подбор - отложено
- Оплата за услуги разведения - отложено
- Система чата/обмена сообщениями - отложена на этап 2

## Ограничения
- **Юридическое:** Должно соответствовать правилам разведения животных в РФ и 152-ФЗ для персональных данных.
- **Качество данных:** Точность подбора зависит от полноты и проверенности профилей животных.
- **Производительность:** Поиск подбора должен возвращать результаты за <2с.
- **Масштабируемость:** Поддержка подбора по 50k+ животных, способных к размножению.
- **Технологии:** Должно соответствовать выбранному стеку (NestJS, TypeScript, PostgreSQL).
- **Удобство использования:** Интерфейс должен быть интуитивно понятен для заводчиков (может не быть техническими экспертами).

## Предыдущие решения
- Подбор изначально будет основываться на поисковых критериях, а не на сложных алгоритмах.
- Мы будем использовать ту же инфраструктуру объявлений/модерации для объявлений о разведении (тип объявления = "разведение").
- Домен животного будет хранить атрибуты, значимые для разведения: ID родословной, результаты тестов на здоровье, титулы и т.д.
- Конфиденциальность: информация о контакте владельца доступна только после того, как модератор одобрит объявление о разведении.
- Подбор на основе местоположения использует тот же гео-поиск, что и домены рынка.
- Мы различаем "объявления о разведении" (животные для спаривания) и "объявления о продаже" (потомство для продажи).

## Трассируемость НФТ
Эта спецификация охватывает следующие нефункциональные требования:
- **Производительность (НФТ-ПРОИЗВ)**: Поиск подбора с 50k животными возвращает результат за <1с для 95% запросов (см. docs/02-requirements/nfr/performance.md)
- **Безопасность (НФТ-БЕЗОП)**: Соблюдение российских правил разведения животных; персональные данные владельца защищены до момента после модерации (см. docs/02-requirements/nfr/security.md)
- **Доступность (НФТ-ДОСТУП)**: Интерфейс подбора следует рекомендациям WCAG 2.1 AA (см. docs/02-requirements/nfr/accessibility.md)

## User Stories

### Управление разведением и подбором
**UC-MT-01:** Как заводчик или фермер, я хочу легко находить подходящих партнёров для разведения моих животных, чтобы я мог улучшить свою программу разведения и генетическое разнообразие.
- Критерии приёмки:
  - Поиск подбора загружается за <2с
  - Фильтры поиска включают вид, породу, возраст, пол, состояние здоровья, родословную, местоположение и предпочтения владельца
  - Поиск на карте с селектором радиуса (1-100 км)
  - Результаты поиска отображают ключевые атрибуты разведения (родословная, сертификаты здоровья, выставочные титулы)
  - Чёткое указание качества подбора/оценки совместимости
  - Возможность сохранять поиски и настраивать оповещения о новых совпадениях
  - Раскрытие прямых контактов после модерации (телефон, ссылки Telegram/VK)
  - Опция преобразования профиля животного в объявление о разведении в один клик

**UC-MT-02:** Как пользователь, заботящийся о приватности и контроле, я хочу управлять видимостью моего животного в поиске для разведения, чтобы я мог защитить свою информацию и соблюдать личные предпочтения.
- Критерии приёмки:
  - Чёткий переключатель для включения/исключения животного из результатов поиска для разведения
  - Визуальная индикация, когда животное скрыто из поиска
  - Простой процесс снова сделать животное видимым
  - Контроль над тем, какая информация о разведении видна (родословная, тесты здоровья, титулы)
  - Возможность заблокировать определённых пользователей от просмотра животного в поиске
  - Журнал аудита того, кто просматривал профиль животного (будущее улучшение)

**UC-MT-03:** Как модератор, я хочу эффективно проверять объявления о разведении, чтобы я мог обеспечить качество, соответствие требованиям и безопасность на рынке разведения.
- Критерии приёмки:
  - Очередь модерации загружается за <2с с чёткими индикаторами статуса
  - Детали объявления о разведении показывают все необходимые атрибуты для проверки (родословная, сертификаты здоровья, титулы)
  - Одобрение/отклонение в один клик с необязательными комментариями
  - Опции массовой модерации для похожих объявлений
  - Чёткий чек-лист соответствия российским правилам разведения животных
  - Система уведомлений для пользователей при изменении статуса объявления
  - Журнал аудита действий модерации для подотчётности
  - Возможность запросить дополнительную информацию у продавца

**UC-MT-04:** Как торговец скотом или фермер, я хочу отслеживать важную информацию о здоровье и родословной, чтобы я мог принимать обоснованные решения о потенциале разведения и ценности моих животных.
- Критерии приёмки:
  - Структурированный ввод тестов здоровья, вакцинаций, лечения
  - Отслеживание родословной со ссылками на мать/отца
  - Сводка о здоровье и родословной видна с первого взгляда в профиле животного
  - Интеграция с медицинскими записями животного из домена животного
  - Возможность экспорта записей о здоровье и родословной (будущее улучшение)
  - Напоминания о предстоящих проверках здоровья или лечении (будущее улучшение)

**UC-MT-05:** Как пользователь, новичок в разведении, я хочу получить доступ к образовательным ресурсам и руководствам, чтобы я мог принимать обоснованные решения о практиках разведения животных.
- Критерии приёмки:
  - Доступ к руководствам по разведению и лучшим практикам
  - Информация о российских правилах и требованиях к разведению
  - Рекомендации по тестированию здоровья для разных видов
  - Объяснение родословной и её важности
  - Ссылки на ветеринарные ресурсы и ассоциации заводчиков
  - Раздел FAQ, охватывающий распространённые вопросы о разведении

## Разбивка задач
1. **Бэкенд (NestJS)**
   - [ ] Расширить сущность Animal полями, специфичными для разведения: pedigreeId, healthTestResults (JSONB), showTitles, breedingRestrictions и т.д.
   - [ ] Убедиться, что тип объявления ListingType включает "разведение" как допустимый тип
   - [ ] Создать сервис поиска подбора, расширяющий гео-поиск фильтрами для разведения
   - [ ] Создать MatchingController (поиск, просмотры профилей животных)
   - [ ] Создать MatchingService (логика бизнеса: алгоритмы поиска, фильтрация)
   - [ ] Настроить индексы базы данных для полей, специфичных для разведения
   - [ ] Написать модульные и интеграционные тесты для функционала поиска/подбора
   - [ ] Создать документацию OpenAPI для конечных точек подбора

2. **Фронтенд (React)**
   - [ ] Создать страницу профиля животного, оптимизированную для отображения при разведении
   - [ ] Создать страницу поиска для разведения с фильтрами (вид, порода, здоровье, местоположение, родословная)
   - [ ] Создать поток создания объявления о разведении (похож на другие типы объявлений, но с полями, специфичными для разведения)
   - [ ] Реализовать выбор местоположения на карте для поиска
   - [ ] Создать компонент карточки животного для результатов поиска разведения
   - [ ] Интегрировать с доменом идентификации для отображения ограниченной информации о владельце после модерации
   - [ ] Написать модульные и end-to-end тесты для потоков работы с подбором

3. **Инфраструктура**
   - [ ] Настроить схему Prisma для полей животного, специфичных для разведения
   - [ ] Убедиться, что индексы базы данных поддерживают комбинированный гео + поиск по атрибутам
   - [ ] Настроить кэширование для частых поисковых запросов на подбор
   - [ ] Реализовать ограничение частоты запросов для конечных точек поиска подбора

## Критерии верификации
- [ ] Модульные тесты >90% покрытия для функционала, связанного с подбором (backend)
- [ ] Интеграционные тесты покрывают: поиск для разведения (различные фильтры), отображение профиля животного, создание объявления о разведении
- [ ] End-to-end тесты покрывают: пользователь ищет партнера для разведения, просматривает профиль животного, создает объявление о разведении
- [ ] Ручное тестирование: проверка точности поиска, валидация, специфичная для разведения, рабочий процесс модерации
- [ ] Производительность: поиск для разведения с 50k животными возвращает результат за <1с для 95% запросов
- [ ] Соответствие: соблюдение российских правил разведения животных; персональные данные владельца защищены до момента после модерации
- [ ] Документация: спецификация OpenAPI сгенерирована и доступна
- [ ] Трассируемость НФТ: проверить, что требования производительности, безопасности и доступности корректно учтены и документированы
