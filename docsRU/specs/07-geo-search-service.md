# Спецификация: Сервис гео-поиска

## Результат
Обеспечить эффективные возможности гео-поиска для поиска животных и объявлений в указанном радиусе (1-100 км) от местоположения пользователя. Поддерживать точные расчеты расстояния и фильтрацию, чтобы обеспечить поиск на основе местоположения во всех доменах рынка (Домашние животные, Скот, Подбор).

## Область и границы
**Включает:**
- Расчет расстояния с использованием формулы Хаverseине или расширения PostGIS
- Поиск по радиусу: найти все точки в пределах X километров от заданной широты/долготы
- Интеграция с сущностями Объявлений (Рынок домашних животных, Рынок скота)
- Интеграция с сущностями Животных (для домена Подбора)
- Поддержка как точного匹 соответствия местоположения, так и поиска по радиусу
- Возможность комбинировать гео-поиск с другими фильтрами (вид, порода, цена и т.д.)
- Оптимизация производительности для больших наборов данных (100k+ записей)
- Конфигурация ограничений радиуса поиска (мин 1км, макс 100км согласно требованиям)

**Исключает:**
- Маршруты/направления - отложено
- Автозаполнение мест/геокодинговое обслуживание (обрабатывается внешним API Карт)
- Сложные поиски по полигонам (например, поиск внутри границ города) - отложено
- Отслеживание местоположения в реальном времени - отложено
- Офлайн карты/кэширование - отложено

## Ограничения
- **Юридическое:** Должен использоваться соответствующий геокодинговый сервис (API Яндекс.Карт для MVP согласно решению о стеке технологий).
- **Точность:** Расчеты расстояния должны быть достаточно точными для доверия пользователей (ошибка в пределах 100м допустима).
- **Производительность:** Запросы гео-поиска должны выполняться за <1s для 95% запросов при ожидаемой нагрузке.
- **Масштабируемость:** Должен эффективно обрабатывать 100k+ записей с геотегами.
- **Технологии:** Использовать PostgreSQL с расширением PostGIS или реализовать эффективную формулу Хаverseине в SQL.
- **Удобство использования:** Радиус поиска должен быть регулируемым через UI (ползунок/ввод) с четкими единицами расстояния (км).
- **Данные:** Данные о местоположении должны храниться как координаты широты/долготы (WGS84).

## Предыдущие решения
- Хранить местоположение как отдельные столбцы с плавающей точкой широты и долготы в таблицах Объявления и опционально Животных.
- Использовать PostgreSQL с расширением earthdistance cube или PostGIS для гео-индексирования (решение будет принято на основе тестирования производительности).
- Для MVP реализовать гео-поиск с использованием формулы Хаverseине, оптимизированной с предварительной фильтрацией ограничивающего прямоугольника для снижения вычислительной нагрузки.
- Внешний геокодинг (адрес в координаты) будет обрабатываться через API Яндекс.Карт via абстракцию frontend/backend.
- Максимальный радиус поиска применяется на уровне 100км для предотвращения злоупотребляющих запросов.
- Минимальный радиус поиска составляет 1км для обеспечения значимых результатов.
- Данные о местоположении обязательны для всех объявлений; пользователи должны предоставлять местоположение через взаимодействие с картой или ввод адреса.

## Разбивка задач
1. **Бэкенд (NestJS)**
   - [ ] Создать共享 сервис `geo-search` (может быть в `src/lib/` или как утилита)
   - [ ] Реализовать функцию расчета расстояния Хаverseине (TypeScript)
   - [ ] Создать конструктор database-запросов, который добавляет гео-фильтр с оптимизацией ограничивающего прямоугольника
   - [ ] Интегрировать гео-поиск в методы поиска Объявлений в сервисах PetMarketplaceService и LivestockMarketplaceService
   - [ ] Интегрировать гео-поиск в методы поиска Животных в сервисе MatchingService/любой другой службе, требующей поиска по местоположению
   - [ ] Добавить валидацию для диапазонов широты/долготы (-90 до 90, -180 до 180)
   - [ ] Добавить валидацию для радиуса поиска (1-100 км)
   - [ ] Создать индексы базы данных для столбцов широты/долготы (рассмотреть составной индекс)
   - [ ] Написать модульные тесты для расчета расстояния и построения запросов
   - [ ] Написать интеграционные тесты для гео-поиска с образцовыми данными
   - [ ] Создать документацию OpenAPI, демонстрирующую параметры гео-поиска в конечных точках объявления/животного

2. **База данных**
   - [ ] Добавить столбцы `широта` и `долгота` в таблицу Объявления (уже есть в схеме)
   - [ ] Рассмотреть добавление столбцов `широта` и `долгота` в таблицу Животного, если необходимо для подбора (опционально)
   - [ ] Создать индексы: CREATE INDEX ON listings USING GIST (ll_to_earth(широта, долгота)); если используется PostGIS
   - [ ] Или создать btree индексы на широте, долготе для предварительной фильтрации ограничивающим прямоугольником
   - [ ] Заполнить некоторые тестовые данные с известными расстояниями для валидации

3. **Фронтенд (React)**
   - [ ] Создать переисползуемый компонент гео-поиска (выбор на карте + селектор радиуса)
   - [ ] Интегрировать с API Яндекс.Карт для поиска по адресу и обратного геокодинга
   - [ ] Создать ввод радиуса (ползунок или числовой ввод) с единицами км
   - [ ] Убедиться, что параметры гео-поиска передаются в API вызовы для поиска объявлений/животных
   - [ ] Отображать расстояние от пользователя в результатах поиска (опционально)
   - [ ] Написать модульные и end-to-end тесты для компонента гео-поиска

4. **Инфраструктура**
   - [ ] Решить, использовать ли PostGIS vs earthdistance vs собственный Хаverseине в зависимости от тестирования производительности
   - [ ] Если используется PostGIS: активировать расширение в PostgreSQL и скорректировать схему
   - [ ] Настроить pooling подключений и таймауты запросов для операций с интенсивным гео-поиском
   - [ ] Рассмотреть кэширование частых результатов гео-поиска (например, популярных местоположений) в Redis

## Критерии верификации
- [ ] Модульные тесты >90% покрытия для сервиса гео-поиска (backend)
- [ ] Интеграционные тесты проверяют: точность расчета расстояния, оптимизацию ограничивающим прямоугольником, корректность поиска по радиусу
- [ ] End-to-end тесты покрывают: пользователь ищет объявления в пределах 5км, видит корректные результаты, регулировка радиуса работает
- [ ] Ручное тестирование: проверка точности по известным расстояниям (например, с использованием инструмента измерения расстояний Google Maps)
- [ ] Производительность: гео-поиск с 100k объявлениями возвращает результат за <1s для 95% запросов при 50 RPS
- [ ] Соответствие: использует API Яндекс.Карт для геокодинга, как указано в решении о стеке технологий
- [ ] Документация: спецификации API четко показывают параметры широты/долготы/радиуса
