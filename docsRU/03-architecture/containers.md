# Контейнеры (C4 уровень 2)

Этот документ описывает контейнеры в системе ZooLink на уровня C2 модели C4.

## Обзор

На уровне контейнеров мы показываем высокоуровневую технологическую структуру системы, включая:
- Веб-приложения
- Мобильные приложения 
- Базы данных
- Файловые хранилища
- Кеш
- Внешние зависимости

## Веб-приложение (Backend)

**Технология:** NestJS (TypeScript) с микросервисной архитектурой модульного монолита

**Ответственное за:**
- RESTful API для всех доменов
- Аутентификация и авторизация
- Бизнес-логика доменов
- Обработка событий и уведомлений
- Интеграция с внешними сервисами
- Модерационный рабочий процесс

**Ключевые характеристики:**
- Модульная структура с четкими границами доменов
- Поддержка TypeORM для работы с БД
- Redis для кеширования и pub/sub
- Архитектура, готовая к микросервисам
- Обработка файлов через S3-совместимое хранилище

## Веб-приложение (Frontend)

**Технология:** React с TypeScript и современным UI-фреймворком

**Ответственное за:**
- Пользовательский интерфейс для всех функций платформы
- Управление состоянием клиента
- Локализация и интернационализация
- Доступность (a11y)
- Адаптивный дизайн

**Ключевые характеристики:**
- Component-based архитектура
- TypeScript для типовой безопасности
- Модульная структура по доменам
- Поддержка PWA возможностей
- Интеграция с картографическими сервисами

## База данных

**Технология:** PostgreSQL с расширением PostGIS

**Ответственное за:**
- Хранение всех сущностей системы (пользователи, животные, объявления и т.д.)
- Геопространственные запросы для поиска по радиусу
- Обеспечение ссылочной целостности
- Аудиторские журналы и исторические данные
- Резервное копирование и восстановление

**Ключевые характеристики:**
- Нормализованная схема данных с JSONB для расширяемости
- Геоиндексы для эффективного поиска по расположению
- Триггеры для автоматического обновления временных меток
- Ограничения для обеспечения бизнес-правил
- Партиционирование для больших таблиц (планируется)

## Кеш

**Технология:** Redis

**Ответственное за:**
- Кеширование часто запрашиваемых данных
- Сессии пользователей
- Очереди задач и фоновая обработка
- Rate limiting
- Временные данные модерации

**Ключевые характеристики:**
- In-memory хранилище с persistence
- Pub/sub для событий в реальном времени
- TTL для автоматического очистки данных
- Кластерная конфигурация для масштабирования

## Хранилище файлов

**Технология:** S3-совместимое хранилище (MinIO для разработки, AWS S3 для production)

**Ответственное за:**
- Хранение фотографий объявлений
- Аватары пользователей
- Медиа-файлы (документы, видео)
- Резервные копии

**Ключевые характеристики:**
- Предварительно подписанные URL для безопасного доступа
- Серверное шифрование
- Политики управления жизненным циклом
- CDN интеграция для доставки контента
- Репликация между регионами (для production)

## Внешние системы (интеграции)

### Аутентификация
- Google OAuth 2.0
- Apple Sign In
- Telegram Login
- VK Login
- SMS шлюз (Twilio или аналогичный)

### Геокартирование
- Yandex.Maps API (основной)
- Резервный: упрощенное расстояние город-город
- Планируемый переход на OSM/PostGIS

### Email сервис
- SendGrid или аналогичный (только транзакционные письма)

### Будущие интеграции (Фаза 2+)
- Платежный шлюз (Stripe, PayPal, Yandex.Kassa)
- Расширенные геокартирование сервисы
- Ветеринарные и медицинские сервисы
- Сервисы логистики и транспорта
- Сервисы генетического тестирования
- Регулятивные системы (Меркурий/ВетИС)
- Социальные медиа платформы
- Сервисы контента и медиа
- Аналитика и мониторинг
