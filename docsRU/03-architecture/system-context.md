Показывает систему ZooLink как единый блок, с её пользователями и внешними системами, с которыми она прямо взаимодействует.

## Описание элементов

### Пользователи
- **Владельцы домашних животных**: Люди, желающие купить, продать или разводить компаньон-животных (кошек, собак, птиц и т.д.).
- **Заводчики**: Профессиональные или любительские заводчики, ищущие партнеров для вязки или продающие потомство.
- **Фермеры/Ранчеры**: Владельцы сельскохозяйственного скота (коровы, лошади, овцы, козы, свиньи, птица) для продажи, разведения или аренды.
- **Модераторы**: Доверенные пользователи, отвечающие за проверку объявлений перед публикацией.
- **Администраторы**: Администраторы платформы, управляющие конфигурацией системы, ролями и интеграциями.

### ZooLink Система
- **Веб Приложение**: Одностраничное приложение, доступное через браузеры, с возможностями PWA для оффлайн кэширования и установки.
- **RESTful API**: Бэкенд-сервис, построенный на NestJS, предоставляющий конечные точки для всей функциональности.
- **PostgreSQL База данных**: Основное хранилище данных для профилей пользователей, животных, объявлений, журналов модерации и справочных данных.
- **Redis Кэш**: Используется для хранения сессий, кэширования справочных данных и временных данных.

### Внешние Системы
- **SMS Шлюз**: Сервис для отправки кодов подтверждения при аутентификации по телефону.
- **Email Сервис**: Сервис для отправки транзакционных писем (подтверждение, результаты модерации, уведомления).
- **Геокодирование и Карты**: Сервис для преобразования адресов в координаты и вычисления расстояний для гео-поиска.
- **Объектное Хранилище**: Масштабируемое хранилище для загруженных пользователями изображений (фото животных, фото объявлений, аватары).
- **OAuth Провайдеры**: Сторонние провайдеры идентификации, обеспечивающие возможность входа через социальные сети.

## Интерфейсы
- **Пользователь ↔ ВебПриложение**: HTTPS доступ через настольный или мобильный браузер.
- **ВебПриложение ↔ API**: REST/JSON через HTTPS.
- **API ↔ База данных**: SQL запросы через Prisma ORM (протокол PostgreSQL).
- **API ↔ Кэш**: Протокол Redis (TCP).
- **API ↔ SMS Шлюз**: HTTPS API (Twilio REST или аналогичный).
- **API ↔ Email Сервис**: HTTPS API (SendGrid v3 или аналогичный).
- **API ↔ Геокодирование/Карты**: HTTPS API (Yandex.Maps Геокодер и Поиск).
- **API ↔ Объектное Хранилище**: S3-совместимый REST API (через предварительно подписанные URL или прямой SDK).
- **API ↔ OAuth Провайдеры**: OAuth 2.0 потоки (authorization code grant с PKCE где применимо).

## Предположения
- ВебПриложение отвечает за весь рендеринг UI; API является纯но данными/сервисами.
- Аутентификационные токены (JWT) управляются ВебПриложением и передаются вместе с запросами к API.
- Загрузка файлов происходит напрямую от ВебПриложения к Объектному Хранилищу с использованием предварительно подписанных URL (API не проксирует данные файлов).
- Все коммуникации зашифрованы (HTTPS/TLS 1.2+).
- Система не интегрируется напрямую с аппаратными устройствами (например, считывателями микрочипов) в MVP.
