## Как использовать эту документацию

1. **Начните с Формулировки проблемы**: Поймите, почему мы создаем ZooLink.
2. **Изучите Краткое описание проекта**: Контекст из фазы инициации.
3. **Ознакомьтесь с Объемом MVP**: Знайте точно, что мы создаем в первом релизе.
4. **Погрузитесь в Требования доменов**: Поймите детали каждого ограниченного контекста.
5. **Изучите Нефункциональные требования**: Безопасность, производительность, ограничения доступности.
6. **Изучите Архитектуру**: Узнайте, как структурирована система и как взаимодействуют компоненты.
7. **Изучите Решения по архитектуре**: Поймите принятые компромиссы.
8. **Посмотрите на UI/UX и Операции**: Для деталей по дизайну и развертыванию.

## Соглашения

- Все документы написаны в формате Markdown.
- Диаграммы используют синтаксис Mermaid, где это уместно.
- Контракты API находятся в формате OpenAPI 3.0 YAML.
- Термины определены согласованно во всех документах.
