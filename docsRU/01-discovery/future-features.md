# Будущие функции: ZooLink

## Цель
Документирует функции и улучшения, запланированные для пост-MVP фаз (Фаза 2+). Это помогает сохранить область MVP фокусированной, сохраняя идеи для будущей разработки.

## Фаза 2: Рост (6-12 месяцев)
### Основные функции
- **Чат в реальном времени**: Текстовый, голосовой и видео чат между пользователями (после показа контактов или внутри объявлений)
- **Многоязычная поддержка**: Интерфейс и контент доступны на нескольких языках (начиная с английского)
- **Видео контент**: 
 - Загрузка видео для объявлений (показать животных в движении)
 - Прямая трансляция для событий (аукционы, выставки)
 - Видео профили для заводчиков/ферм
- **Продвинутый поиск**:
 - Сохраненные поиски с оповещениями
 - Поиск по изображению (найти похожих животных)
 - Поиск по генетическим признакам
- **Репродуктивные инструменты**:
 - Отслеживание и предсказание течки
 - Календарь спаривания с напоминаниями
 - Отслеживание беременности и расчет даты родов
- **Здоровье и генетика**:
 - Цифровой паспорт здоровья (записи о вакцинации, результаты тестов)
 - Портал генетики (результаты ДНК тестов, анализ родословной)
 - Система оповещений о здоровье (вспышки, напоминания о вакцинации)
- **Социальные функции**:
 - Подписка пользователей/подписчики
 - Лента активности (что делают пользователи, на которых вы подписаны)
 - Группы и сообщества (по породе, интересу, местоположению)
 - Форум для обсуждений и вопросов и ответов
- **Контент и образование**:
 - Платформа публикации статей (советы, руководства, новости)
 - Энциклопедия пород
 - Вебинары и онлайн курсы
- **Монетизация (мягкий запуск)**:
 - Промоутинг объявлений (платить за более высокое появление в поиске)
 - Премиум профили (верифицированные значки, улучшенные галереи)
 - Генерация лидов для ветеринаров и сервисов
 - Партнерский маркетинг для товаров для животных и сельского хозяйства

### Технические улучшения
- **Производительность**:
 - Читающие реплики для базы данных
 - Продвинутое кэширование (кластер Redis, CDN для изображений)
 - Сервис оптимизации изображений (изменение размера и сжатие на лету)
- **Масштабируемость**:
 - Микросервисы для компонентов с высоким трафиком (сопоставление, уведомления)
 - Очередь сообщений (RabbitMQ/Amazon SQS) для асинхронной обработки
 - Оркестрация Kubernetes
- **Поиск**:
 - Elasticsearch или аналогичный для продвинутого текстового и гео-поиска
 - Фасетный поиск и улучшения автодополнения
- **Безопасность**:
 - Веб-прикладной фаервол (WAF)
 - Регулярное тестирование на проникновение
 - Улучшенный мониторинг и оповещение
 - MFA для всех пользователей (опционально)
- **Данные и аналитика**:
 - Хранилище данных для бизнес-аналитики
 - Отчеты по рыночной разведке (агрегированные, анонимизированные)
 - Framework A/B тестирования
 - Аналитика поведения пользователей

### Регуляторное соответствие и соблюдение норм
- **Регуляторная интеграция**:
 - Подготовка к интеграции Меркурий/ВетИС (отслеживание перемещения сельскохозяйственных животных)
 - Автоматическая генерация документов (договоры купли-продажи, справки о здоровье)
- **Доступность**:
 - Улучшенная поддержка скринридеров
 - Учет жестового языка для видео контента
 - Настраиваемый UI (контраст, размеры шрифтов)
- **Юридическое**:
 - Обновление Условий использования и Политики конфиденциальности для новых функций
 - Система управления согласием

## Фаза 3: Зрелость (12+ месяцев)
### Основные функции
- **Полная поддержка транзакций**:
 - Сервис эскроу для высокозначных транзакций
 - Интегрированные платежи (безопасная оплата)
 - Координация доставки и логистики
- **Продвинутые инструменты разведения**:
 - Конструктор и генератор родословной
 - Калькулятор коэффициента близкородового скрещивания
 - Интеграция оценочных племенных значений (EBV)
 - Торговля эмбрионами и ооцитами
- **ИИ и машинное обучение**:
 - Автоматическая помощь в модерации объявлений
 - Движок предложений цен (на основе исторических данных)
 - Улучшение рекомендаций по подбору (на основе ML)
 - Модерация изображений (обнаружение неподобающего контента)
 - Распознавание пород по фото
- **IoT и умное фермерство**:
 - Интеграция с программным обеспечением для управления фермой
 - Отображение данных сенсоров (температура, активность и т.д.)
 - Оповещения об автоматическом обнаружении течки
- **Расширение маркетплейса**:
 - Полная электронная коммерция для снабжения (корм, оборудование, лекарства)
 - Сервисовый маркетплейс (вет, транспорт, обучение)
 - Платформа аукционов ( timed и live)
- **Глобализация**:
 - Поддержка нескольких валют
 - Учет международной доставки
 - Соблюдение нормативных требований в нескольких странах
- **Сообщество и события**:
 - Календарь событий (выставки, продажи, семинары)
 - Продажа билетов на события
 - Каталоги участников и нетворкинг

### Технические улучшения
- **Архитектура**:
 - Сервисная сеть для связи микросервисов
 - Платформа потоковой передачи событий (Kafka) для потоков данных в реальном времени
 - Продвинутые стратегии кеширования (многоуровневые, предсказательные)
- **Наука о данных**:
 - Прогностическая аналитика (вспышки заболеваний, рыночные тренды)
 - Персонализированные рекомендации
 - Обработка естественного языка для поиска и контента
- **Безопасность**:
 - Архитектура zero trust
 - Продвинутое обнаружение и реагирование на угрозы
 - Регулярные сторонние аудиты безопасности
- **DevOps**:
 - Продвинутый CI/CD с сине-зеленым развертыванием
 - Канареечные выпуски и feature flags в масштабе
 - Всеобъемлемая наблюдаемость (логи, метрики, трассировки)

## Невходит в область Фазы 2&3 (Идеи для далекого будущего)
- Просмотр животных в виртуальной реальности
- Консультации по генетическому редактированию (этические соображения)
- Интеграция с метавселенной для виртуальных выставок
- Автономные транспортные средства для перевозки скота
- Блокчейн для записей о родословной и здоровье (если технология созреет)

## AI-эксплуатация площадки (долгосрочное видение — см. ADR-0006)

Стратегическое направление (ADR-0006): роли операторов — сначала **Модератор**, в перспективе **Админ** — могут выполняться специально обученными **ИИ-агентами**, с движением к механизму, который **ведёт и сопровождает площадку как бизнес преимущественно силами ИИ-агентов**, при людях в ролях управления/подотчётности.

- **Точка входа в Фазе 2**: ИИ-ассистируемая модерация (human-in-the-loop) — агент предлагает APPROVE/REJECT/CHANGES_REQUESTED с оценкой уверенности; человек подтверждает. (Согласуется с пунктом «автоматическая модерация (Phase 2)» в домене модерации.)
- **Постепенная автономия**: ассистируемая → поднадзорная (автономно выше порога уверенности, низкая — эскалация людям) → операционные агенты для админа/справочных данных/ops → AI-эксплуатация бизнеса.
- **Заложено в модель данных уже сейчас**: `users.principal_type` (`HUMAN`/`AGENT`) позволяет агенту занимать операторские роли; все действия агента пишутся в неизменяемый аудит `moderation_decisions`; агенты неактивны до включения feature-флагом.
- **Неотъемлемые условия**: ответственным остаётся человек/юрлицо (152-ФЗ, запрещённый контент); least-privilege учётные данные агента; обратимость и человеческое переопределение.
- **Знания**: агенты могут использовать RAG/RLM поверх документации (см. `RLM_RAG_HANDOFF.md`).

## Как использовать этот документ
- Просматривайте во время подготовки бэклога и планирования спринтов
- Функции переходят в область MVP только через формальный Запрос на изменение
- Приоритизируйте на основе обратной связи от пользователей, бизнес-целей и технической осуществимости
- Удаляйте функции, которые больше не являются актуальными или заменены лучшими идеями
- Архивируйте реализованные функции в отдельный журнал "Released Features"
