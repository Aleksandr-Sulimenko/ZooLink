# Домен Маркетплейса для Домашних Животных: ZooLink

## Цель
Обрабатывает объявления для сопроводительных животных (домашних питомцев), таких как кошки, собаки, птицы, кролики, рептилии и мелкие млекопитающие. Фокусируется на некоммерческих транзакциях: продажа, усыновление, услуги спаривания и передержка. Подчеркивает темперамент, здоровье и пригодность для домашней среды вместо продуктивных качеств.

## Основные концепции
- **Объявление о домашнем животном**: Реклама для транзакции, связанной с домашним животным. Типы:
 - `SALE`: Передача собственности за компенсацию или бесплатно
 - `MATING`: Предложение услуг производителя или поиск партнера для разведения
 - `ADOPTION`: Бесплатная передержка (часто в контексте спасения/приюта)
 - `STUD_SERVICE`: Платная услуга производителя (животное мужского пола доступно для разведения)
- **Атрибуты домашнего животного**: Характеристики, относящиеся к сопровождению и разведению (темперамент, здоровье, основы родословной).
- **Жизненный цикл объявления**: От черновика к publикации через премодерацию, затем к активному/завершенному/архивированному.
- **Географический охват**: Поиск по радиусу (1-100 км) критичен для транзакций с домашними животными из-за стресса при транспортировке и предпочтения владельцев местного самовывоза.

## Бизнес-правила
### 1. Создание объявления
- Только аутентифицированные пользователи могут создавать объявления.
- Каждое объявление должно быть связано с **одним** животным из należących пользователя (см. Домен Животных).
- Обязательные поля при создании:
 - `animal_id` (ссылка на Домен Животных)
 - `listing_type` (ПЕРЕЧИСЛЕНИЕ: SALE, MATING, ADOPTION, STUD_SERVICE)
 - `title` (краткий заголовок, максимум 100 символов)
 - `description` (подробный текст, максимум 2000 символов)
 - `price_or_terms`:
 - Для SALE: число (валюта RUB) или "free"/"negotiable"
 - Для MATING/STUD_SERVICE: "free", "pick of litter", фиксированная плата (RUB), или "negotiable"
 - Для ADOPTION: обычно "free" или "donation welcome"
 - `location` (город + необязательная точность адреса; обязательно для модерации и гео-поиска)
 - `photos` (1-5 изображений; минимум 1 четкая фотография животного)
 - `contact_method_visibility` (логическое значение: показывать ли телефон/соцсети после модерации)
- **Присваивание объявления организации/филиалу**:
   - При создании объявления пользователь должен указать либо:
     - Свой персональный аккаунт (через `creator_id`) **ИЛИ**
     - Организацию (через `organization_id`) и опционально филиал (через `branch_id`).
   - `creator_id` (тот человек, который подал объявление) всегда записывается в целях аудита.
   - Объявления, привязанные к организации, показывают название организации (и филиал, если указан) в публичном виде.
   - **Важно**: При размещении объявления от имени организации животное должно принадлежать этой организации (т.е. `organization_id` животного должно совпадать с `organization_id` объявления).
 - Опциональные, но рекомендуемые поля:
 - Статус вакцинации (выпадающий список: актуальный, частичный, отсутствует + даты, если известны)
 - Дескрипторы темперамента (чек-лист: дружелюбен с детьми, дружелюбен с собаками, дружелюбен с кошками, энергичный, спокойный, тревожный, защитный)
 - Заметки о здоровье (свободный текст: стерилизован/кастрирован, известные аллергии, хронические состояния)
 - Информация о родословной (текст: зарегистрировано в клубе? номер регистрации, если доступен)
 - Для MATING/STUD_SERVICE: 
 - Даты течки самки (если применимо)
 - Доказанный статус производителя самца (количество пометов, ветеринарные сертификаты)
 - Условия: естественное спаривание vs. ИИ, место спаривания, условия оплаты производителя

### 2. Валидация и модерация объявлений
- Все объявления поступают в состояние `PENDING_MODERATION` после отправки.
- Проверки модетора:
 - **Аутентичность**: Фотографии соответствуют объявленному животному (вид/порода/пол); нет стоковых изображений.
 - **Полнота**: Все обязательные поля заполнены; описание не спам.
 - **Соответствие политике**: 
 - Нет продвижения запрещенных действий (например, породы собак для боевых действий в контекстах, подразумевающих насилие).
 - Нет вводящих в заблуждение заявлений (например, "чистокровный" без документов при заявлении о чемпионатных линиях).
 - Заявления о вакцинации вызывают умеренное доверие, но не проверяются на MVP (честность пользователя).
 - **Обнаружение дубликатов**: Предупреждает, если существует почти идентичное объявление от того же пользователя недавно (возможная ошибка).
- Действия модетора:
 - `APPROVE`: Объявление становится `PUBLISHED` и видимо в поиске.
 - `REJECT`: Возвращается в состояние `DRAFT` с необходимыми комментариями правки; пользователь может повторно отправить.
- Время на модерацию: Цель <4 часов в рабочие часы (9AM-9PM местного времени).

### 3. Состояния жизненного цикла объявления
- `DRAFT`: Редактируемый пользователем, не отправлен.
- `PENDING_MODERATION`: На ожидании рассмотрения.
- `PUBLISHED`: Активен в поиске; может получать просмотры/запросы контактов.
- `CONTACTED`: Отслеживаемое системой состояние, когда показаны контакты (не влияет на видимость).
- `COMPLETED`: Пользователь помечает как успешную транзакцию (продажа/спаривание occurred).
- `ARCHIVED`: Пользователь скрывает объявление (сохраняет историю; может быть реактивировано).
- `EXPIRED`: Автоматически через 60 дней, если не завершено/архивировано (настраивается по типу).
- Переходы состояний обеспечиваются бэкендом; UI отражает текущее состояние.

### 4. Поиск и обнаружение
- Фильтры поиска для Рынка Домашних Животных:
 - `listing_type` (SALE/MATING/ADOPTION/STUD_SERVICE)
 - `species` (cat/dog/bird/rabbit/reptile/rodent/etc.)
 - `breed` (из справочника; поддерживает смешанное/неизвестное)
 - `sex` (male/female)
 - `age_range` (производится из даты рождения животного)
 - `price_range` (мин/макс в RUB; "free" рассматривается как 0)
 - `location_radius` (от города пользователя; 1-100 км)
 - `temperament_tags` (мультивыбор: kid-friendly, dog-cat-friendly, и т.д.)
 - `health_flags` (вакцинирован, стерилизован/кастрирован, обработан от гельминтов)
 - `has_pedigree_papers` (логическое значение)
- Параметры сортировки:
 - `newest_first` (по умолчанию)
 - `price_low_to_high`
 - `price_high_to_low`
 - `distance_closest`
- Результаты поиска показывают:
 - Миниатюрная фотография
 - Заголовок, вид/порода, пол, индикатор возраста
 - Цена/условия
 - Расстояние от пользователя
 - Значок для подтвержденного производителя (если применимо) или статуса вакцинации
- При нажатии на объявление показывается:
 - Полное описание
 - Все фотографии (карусель)
 - Детали животного (из связанного профиля животного)
 - Заметки о здоровье/темпераменте
 - Город владельца (точный адрес скрыт до показа контактов)
 - Кнопка "Показать контакты" (видна только после одобрения модерации)

### 5. Взаимодействие после модерации
- Когда пользователь нажимает "Показать контакты" на объявлении со статусом `PUBLISHED`:
 - Система фиксирует событие (listing_id, viewer_user_id, timestamp).
 - Раскрывает:
 - Номер телефона (если владелец указал при регистрации и согласился делиться)
 - Ссылки на связанные социальные профили (Telegram, VK) если доступны и согласованы.
 - НЕ раскрывает точный адрес; пользователи договариваются о встрече через раскрытые контакты.
 - Владелец может видеть в аналитике: "Ваше объявление просматривалось X раз, контакты показывались Y раз."

### 6. Особенные правила по типу объявления
- **SALE**:
 - Цена должна быть ≥0, если числовая; "free" или "negotiable" допустимы.
 - Нет ограничения на продажу щенков/котят ниже возрастного предела на MVP (полагается на этику пользователей и выборочные проверки модератора по очевидным проблемам благосостояния).
 - Поощряется упомянуть статус стерилизации/кастрации, если применимо.
- **MATING**:
 - Обе стороны должны указать, что они предлагают (услуга производителя, выбор из помета и т.д.).
 - Рекомендуется обсудить ветеринарное тестирование (бруцеллез, бруцелла и т.д.) вне платформы; система не enforces.
 - Нет гарантии успешной беременности; транзакция считается завершенной после попытки спаривания, если иначе не согласовано.
- **ADOPTION**:
 - Поле цены по умолчанию установлено в "free"; можно предложить пожертвование приюту.
 - Часто включает предысторию: спасен, сдан и т.д.
- **STUD_SERVICE**:
 - Явно для самцов животных, доступных для разведения.
 - Структура оплаты должна быть明确ной (за попытку, за беременность, выбор из помета).
 - Владелец отвечает за проверку здоровья самки перед принятием.

## Нефункциональные требования (специфичные для Рынка Домашних Животных)
- **Производительность**: 
 - Создание объявления: <2с (включает загрузку фото в хранилище).
 - Результаты поиска: <1.5s для 95% запросов (<100км радиус, умеренные фильтры).
 - Загрузка фото: Оптимизированные миниатюры; ленивая загрузка в UI.
- **Масштабируемость**: 
 - Поддержка 10k активных объявлений о домашних животных без замедления.
 - Обработка 500 новых отправок объявлений в день в пик.
- **Согласованность данных**:
 - Объявление должно всегда ссылаться на валидное, активное животное.
 - Если животное деактивировано, объявление показывает предупреждение, но остается активным до действий пользователя.
- **Расширяемость**:
 - Поле JSONB `metadata` для экспериментальных атрибутов (например, ссылки на социальные сети, placeholder для URL видео).
 - Новые типы объявлений могут быть добавлены через расширение ПЕРЕЧИСЛЕНИЯ (обратно совместимо).
- **Безопасность/Конфиденциальность**:
 - Точное местоположение никогда не показывается; толькоdistance/радиус поиска.
 - Информация для связи раскрывается только по действию пользователя (не при наведении/предпросмотре).
 - Ограничение скорости на "Показать контакты": максимум 10 раскрытий в час на пользователя для предотвращения скрейпинга.
- **Нагрузка на модератора**: 
 - Рассчитано на <50 объявлений/день умеренная скорость на MVP.
 - Будущее: Предварительный скрининг с помощью ML для очевидного спама/нерелевантного контента.

## Правила валидации
- `animal_id` должен ссылаться на активное животное, принадлежащее `creator_id`.
- Массив `photos` должен содержать 1-5 элементов; каждый элемент - URL в хранилище.
- `price_or_terms`:
 - Если строковое число, должно быть разбираемым как положительное целое.
 - Свободный текст условий ограничен 100 символами.
- `location_city_id` должен существовать в справочнике.
- `description` не может быть пустым или состоять только из пробелов.
- Для MATING/STUD_SERVICE: если `price_or_terms` подразумевает плату, она должна быть числовой или "negotiable".

## Пользовательский путь: Создание и получение интереса к объявлению о домашнем животном
```mermaid
sequenceDiagram
 участник Пользователь как Владелец
 участник Пользователь как Искатель
 uczestник Frontend
 участник Бэкенд (NestJS Listing Module)
 участник Хранилище (S3)
 участник База данных

 %% Владелец создает объявление
 Владелец->>Frontend: Переходит к "Мои животные" -> выбирает питомца -> "Создать объявление"
 Frontend->>Бэкенд: GET /animals?owner_id=me (для выбора животного)
 Бэкенд->>База данных: Возвращает активные животные владельца
 База данных-->>Бэкенд: Список животных
 Бэкенд->>Frontend: Возвращает список животных

 Владелец->>Frontend: Выбирает животное, заполняет форму (type=SALE, price=5000, описание, фото)
 Frontend->>Хранилище: Загружает фото (подписанные URL)
 Хранилище-->>Frontend: Возвращает URL фотографий
 Frontend->>Бэкенд: POST /listings {animal_id, title, description, price_or_terms, location_city_id, photos:[urls]}
 Бэкенд->>База данных: Валидирует, создает запись объявления (status=DRAFT)
 Бэкенд->>Frontend: Возвращает ID объявления

 Владелец->>Frontend: Нажимает "Отправить на модерацию"
 Frontend->>Бэкенд: PATCH /listings/{id} {status: PENDING_MODERATION}
 Бэкенд->>База данных: Обновляет статус
 Бэкенд->>Frontend: Возвращает успех

 %% Модерация (упрощенно)
 Модератор->>Frontend: Просматривает очередь модерации
 Frontend->>Бэкенд: GET /moderation/queue?type=pet&limit=20
 Бэкенд->>База данных: Возвращает ожидающие объявления о домашних животных
 База данных-->>Бэкенд: Список объявлений
 Бэкенд->>Frontend: Возвращает список

 Модератор->>Frontend: Просматривает объявление, нажимает "Одобрить"
 Frontend->>Бэкенд: PATCH /listings/{id} {status: PUBLISHED, moderation_log: [...]}
 Бэкенд->>База данных: Обновляет статус + журнал
 Бэкенд->>Frontend: Возвращает успех

 %% Искатель находит объявление
 Искатель->>Frontend: Ищет: вид=Dog, порода=Labrador, радиус=20км, max_price=10000
 Frontend->>Бэкенд: GET /listings?вид=Dog&порода=Labrador&location_radius=20&max_price=10000
 Бэкенд->>База данных: Ищет объявления с гео-фильтром + фильтром цены
 База данных-->>Бэкенд: Объявления с миниатюрами
 Бэкенд->>Frontend: Возвращает объявления с миниатюрами
 Frontend->>Искатель: Показывает результаты

 Искатель->>Frontend: Нажимает на объявление -> "Показать контакты"
 Frontend->>Бэкенд: POST /listings/{id}/show-contacts
 Бэкенд->>База данных: Увеличивает contact_shown_count, фиксирует событие
 База данных-->>Бэкенд: Обновленные счетчики
 Бэкенд->>Frontend: Возвращает телефон владельца (если предоставлен) + ссылки на TG/VK
 Frontend->>Искатель: Отображает информацию для связи

 %% Владелец видит аналитику
 Владелец->>Frontend: Просматривает "Мои объявления" -> выбирает объявление -> "Статистика"
 Frontend->>Бэкенд: GET /listings/{id}/analytics
 Бэкенд->>База данных: Возвращает view_count, contact_shown_count, и т.д.
 База данных-->>Бэкенд: Статистика
 Бэкенд->>Frontend: Возвращает статистику
 Frontend->>Владелец: Отображает: "Просмотры: 15, Показано контактов: 3"
```

## Открытые вопросы и предположения
- **Предположение**: Поля темперамента и здоровья основаны на честности пользователей; модератор выполняет выборочные проверки на явные ложные доклады (например, заявление о вакцинированном щенке возрастом 2 недели).
- **Предположение**: Фотографии модерируются по релевантности (должны показывать животное); нет AI-проверки породы на MVP.
- **Открытый вопрос**: Следует ли ограничивать размещение объявлений о щенках/котятах младше 8 недель? (Решено: полагаться на этику пользователей + вмешательство модератора при явных нарушениях благосостояния на MVP; возможно добавить правило в Фазу 2.)
- **Предположение**: Объявления с надписью "free" распространены; система не различает настоящие бесплатные и символические платы (например, 1 RUB).
- **Предположение**: Пользователи понимают, что встреча с домашними животными лично связана с риском; платформа лишь facilitiates знакомство.

## Связанные домены
- **Домен Животных**: Предоставляет основной профиль домашнего животного; объявление ссылается на него.
- **Домен Идентификации**: `creator_id` связывает с пользователем; требуется аутентификация.
- **Домен Администрирования**: Управляет справочными данными (породы, города), очередью модерации, причинами отклонения.
- **Домен Сопоставления**: Использует данные объявлений о домашних животных для предложения совместимых партнеров (фильтрация по течке, темпераменту, родословной).
- **Будущие домены**: Паспорт Здоровья (расширяет данные животных), Календарь Разведения (использует объявления о спаривании).
