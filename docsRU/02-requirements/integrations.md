# Интеграции: ZooLink

## Цель
Документирует внешние системы и сервисы, с которыми ZooLink интегрируется для предоставления основной функциональности. Это включает как интеграции для MVP, так и запланированные будущие интеграции.

## Интеграции MVP
Эти интеграции необходимы для функционирования MVP.

### 1. Провайдеры Аутентификации
#### Google OAuth 2.0
- **Цель**: Позволить пользователям зарегистрироваться/войти с использованием аккаунта Google
- **Обмениваемые данные**: ID пользователя, email, имя, фото профиля (опционально)
- **Безопасность**: Использует поток OAuth 2.0 с параметром state и PKCE
- **Резервный вариант**: Если Google недоступен, пользователь может использовать аутентификацию по телефону/SMS
- **Лимиты запросов**: Подпадает под квоты API Google (отслеживаются)

#### Apple Sign In
- **Цель**: Позволить пользователям зарегистрироваться/войти с использованием Apple ID
- **Обмениваемые данные**: ID пользователя, email (если предоставлен), имя
- **Безопасность**: Использует OAuth 2.0 с JWT
- **Резервный вариант**: Доступны альтернативные методы аутентификации

#### Telegram Login
- **Цель**: Позволить пользователям зарегистрироваться/войти с использованием аккаунта Telegram
- **Обмениваемые данные**: ID пользователя, username, имя, фамилия, URL фото
- **Безопасность**: Использует протокол Login Widget Telegram
- **Резервный вариант**: Доступны альтернативные методы аутентификации

#### VK Login (VKontakte)
- **Цель**: Позволить пользователям зарегистрироваться/войти с использованием аккаунта VK
- **Обмениваемые данные**: ID пользователя, email, имя, фото профиля
- **Безопасность**: Использует реализацию OAuth 2.0 VK
- **Резервный вариант**: Доступны альтернативные методы аутентификации

### 2. SMS Шлюз
- **Провайдер**: **SMS.RU** (РФ по умолчанию; альтернативы SMSC.RU / MTS Exolve — см. ADR-0008). Twilio заблокирован в РФ.
- **Цель**: Отправка кодов подтверждения для аутентификации по номеру телефона
- **Обмениваемые данные**: Номер телефона, код подтверждения
- **Безопасность**: Использует HTTPS с API ключами, хранящимися в управлении секретами
- **Лимиты запросов**: Отслеживается для соблюдения бесплатного тарифа; доступен путь апгрейда
- **Резервный вариант**: Если SMS недоступен, пользователь может использовать провайдеров OAuth

### 3. Геокодирование и Карты
- **Провайдер**: Yandex.Maps API (бесплатный тариф)
- **Цель**: 
 - Преобразование названий городов/регионов в координаты для гео-поиска
 - Вычисление расстояний между пользователями и объявлениями
 - Отображение карт в деталях объявлений
- **Обмениваемые данные**: 
 - Запрос: Компоненты адреса (город, регион)
 - Ответ: Широта/долгота, форматированный адрес
- **Безопасность**: API ключ ограничен определенными реферерами и диапазонами IP
- **Лимиты запросов**: 
 - Бесплатный тариф: 10,000 запросов/день (отслеживается)
 - Превышение бесплатного тарифа: платный тариф или переход на OSM/PostGIS
- **Резервный вариант**: 
 - Для MVP: Использовать упрощенное расстояние город-город (прямая линия) при сбое API
 - В будущем: Перейти на самодостаточный OSM с PostGIS для unlimited использования

### 4. Email Сервис
- **Провайдер**: **Unisender** (РФ по умолчанию; альтернатива Mailopost — см. ADR-0008). SendGrid заблокирован в РФ.
- **Цель**: Отправка только транзакционных писем
 - Подтверждение email (опционально)
 - Результаты модерации (одобрено/отклонено)
 - Сброс пароля (если реализовано)
 - Системные уведомления
- **Обмениваемые данные**: Email получателя, тема, тело (HTML/текст)
- **Безопасность**: API ключ с ограниченными правами (только отправка)
- **Лимиты запросов**: Отслеживается для соблюдения бесплатного тарифа
- **Резервный вариант**: 
 - Для некритических писем: только уведомления в приложении
 - Критические: SMS резервный вариант для кодов подтверждения

### 5. Объектное Хранилище (для Фотографий)
- **Провайдер**: S3-совместимый сервис (MinIO для разработки, AWS S3 или аналогичный для production)
- **Цель**: Хранение и обслуживание фотографий объявлений и аватаров пользователей
- **Обмениваемые данные**: 
 - Загрузка: Предварительно подписанный PUT URL, файл фотографии
 - Извлечение: Прямой GET к URL объекта (через CDN или напрямую)
- **Безопасность**: 
 - Политики бакета: Приватный бакет, доступ только через предварительно подписанные URL
 - Шифрование: Серверное шифрование (SSE-S3) включено
 - Управление доступом: Ограничение CORS только доменами ZooLink
- **Лимиты запросов**: 
 - Запросы: Отслеживаются на предмет злоупотребления
 - Хранилище: Отслеживается на предмет стоимости
- **Резервный вариант**: 
 - При сбое основного хранилища: резервное хранилище в другом регионе
 - Для разработки: локальное хранилище файлов (не для production)

## Будущие Интеграции (Фаза 2+)
Эти интеграции запланированы на более поздние этапы.

### 6. Платежный Шлюз
- **Цель**: Обработка платежей для функций монетизации (Boost, премиум профили, эскроу)
- **Провайдер (РФ по умолчанию)**: **ЮKassa + СБП** (альтернативы Т-Касса / CloudPayments — см. ADR-0008). Stripe/PayPal заблокированы в РФ и не используются. Требуется 54-ФЗ фискализация.
- **Обмениваемые данные**: Детали платежа, метаданные транзакции
- **Безопасность**: Соответствие PCI DSS через сертифицированный шлюз (мы не обрабатываем данные карт напрямую)
- **Интеграция**: API-based, вебхуки для статуса платежа

### 7. Расширенные Сервисы Геокартирования/Геолокации
- **Цель**: Улучшенный гео-поиск, маршрутизация и локационные функции
- **Рассматриваемые провайдеры**: 
 - Google Maps Platform (Directions, Distance Matrix, Places)
 - Mapbox
 - Open-source: OSRM, GraphHopper (самодостаточные)
- **Обмениваемые данные**: Точки отправления, назначения, режим передвижения
- **Сценарии использования**:
 - Расчет времени/расстояния на автомобиле для организации встреч
 - Отображение маршрута на карте
 - Поиск объявлений вдоль маршрута
 - Поиск по зоне обслуживания (для ветеринаров, транспортёров)

### 8. Ветеринарные и Медицинские Сервисы
- **Цель**: Интеграция с ветеринарными клиниками для проверки медицинских записей и записи на прием
- **Рассматриваемые провайдеры**: 
 - Программное обеспечение для управления практикой (например, IDEXX, AVImark)
 - Телемедицинские платформы
 - Порталы результатов лабораторных исследований
- **Обмениваемые данные**: Записи о вакцинации, результаты тестов, планирование встреч
- **Сценарии использования**:
 - Проверка заявлений о вакцинации в объявлениях
 - Позволение пользователям бронировать ветеринарные визиты через платформу
 - Отображение предстоящих медосмотров в профиле животного

### 9. Сервисы Логистики и Транспорта
- **Цель**: Облегчение организации транспортировки животных (особенно для сельскохозяйственных)
- **Рассматриваемые провайдеры**: 
 - Компании по транспортировке скота
 - Общие грузовые API (например, Uber Freight, Convoy)
 - Специализированные платформы транспортировки животных
- **Обмениваемые данные**: Детали самовывоза/доставки, спецификации животных, регулятивные документы
- **Сценарии использования**:
 - Предоставление транспортных расценок
 - Генерация сертификатов здоровья для межштатного перемещения
 - Отслеживание отправлений в реальном времени

### 10. Сервисы Генетического Тестирования
- **Цель**: Позволить пользователям заказывать и просматривать результаты генетических тестов напрямую
- **Рассматриваемые провайдеры**: 
 - Коммерческие лаборатории генетического тестирования (например, Zoetis, Neogen, Embark)
 - Университетские службы расширения
- **Обмениваемые данные**: Заказы тестов, результаты (сырые данные и интерпретации)
- **Сценарии использования**:
 - Отображение генетических панелей здоровья в профилях животных
 - Предложение комплементарных партнеров на основе генетической совместимости
 - Предупреждение пользователей о потенциальных генетических нарушениях

### 11. Регулятивные Системы (Меркурий/ВетИС)
- **Цель**: Соблюдение требований по отчетности о перемещении скота (Фаза 3 для сельскохозяйственных животных)
- **Провайдер**: Российская Федеральная государственная информационная система "Меркурий" (VetIS)
- **Обмениваемые данные**: 
 - Исходящие: Запросы на перемещение скота (заявления на разрешение)
 - Входящие: Подтверждения перемещения, вспышки заболеваний
- **Сценарии использования**:
 - Автоматизация создания разрешений для объявленного скота
 - Проверка того, что покупатели имеют соответствующие приемные помещения
 - Получение предупреждений о региональных вспышках заболеваний
 - Ведение аудиторского traces для проверок соответствия

### 12. Социальные Медиа Платформы
- **Цель**: Улучшение профилей пользователей и обеспечение возможности обмена
- **Рассматриваемые провайдеры**: 
 - Facebook, Instagram, Twitter/X
 - TikTok, YouTube
- **Обмениваемые данные**: 
 - Исходящие: Публикация объявлений, достижений, статей
 - Входящие: Фото профиля, основная информация (через вход)
- **Сценарии использования**:
 - Позволение пользователям делиться своими объявлениями в социальных сетях
 - Импорт фото профиля из социальных аккаунтов
 - Обеспечение возможности входа через соцсети (уже покрыто в интеграциях MVP)
 - Получение контента, связанного с породами, для энциклопедии

### 13. Сервисы Контента и Медиа
- **Цель**: Улучшение доставки контента и пользовательского опыта
- **Рассматриваемые провайдеры**: 
 - Видеостриминг (Vimeo, YouTube API для встроенного контента)
 - Обработка изображений (Cloudinary, Imgix для на лету трансформаций)
 - Генерация документов (Docraft, PDFLib для контрактов/сертификатов)
 - Новостные API (для разделов сельского хозяйства/домашних животных новостей)
- **Сценарии использования**:
 - Оптимизация доставки изображений (изменение размера, формат, сжатие)
 - Генерация договоров купли-продажи и сертификатов здоровья
 - Встраивание образовательных видео в статьи
 - Отображение релевантных новостей в ленте пользователя

### 14. Аналитика и Мониторинг
- **Цель**: Улучшение наблюдаемости и бизнес-аналитики
- **Рассматриваемые провайдеры**: 
 - Мониторинг производительности приложений: **Prometheus + Grafana / VictoriaMetrics** (РФ по умолчанию, self-hostable; см. ADR-0008); Datadog/New Relic только где доступна оплата
 - Агрегация логов: ELK Stack, Splunk
 - Отслеживание ошибок: Sentry, Rollbar
 - Бизнес-аналитика: Tableau, Power BI, или open-source (Metabase, Superset)
- **Обмениваемые данные**: 
 - Исходящие: Метрики, логи, трассировки, события
 - Входящие: Дашборды, оповещения, отчеты
- **Сценарии использования**:
 - Мониторинг здоровья и производительности системы
 - Отслеживание поведения пользователей и воронок конверсии
 - Отладка проблем в production
 - Генерация отчетов для заинтересованных сторон

## Принципы Интеграции
1. **Слабая Связанность**: 
 - Интеграции находятся позади адаптеров или сервисов
 - Позволяет заменять провайдеров без изменения основной логики
2. **Безопасность Превыше Всего**: 
 - API ключи хранятся в управлении секретами (никогда не в коде)
 - Принцип наименьших привилегий для API ключей
 - Регулярная ротация учетных данных
3. **Обработка Ошибок и Устойчивость**:
 - Паттерн circuit breaker для внешних вызовов
 - Механизмы резервирования где возможно
 - Плавная деградация (например, если карты недоступны, использовать более простое расстояние)
4. **Ограничение Скорости и Квоты**:
 - Мониторинг использования относительно лимитов провайдеров
 - Внедрение клиентского троттлинга
 - Оповещения при приближении к лимитам
5. **Конфиденциальность Данных**:
 - Делитесь только необходимыми данными с третьими сторонами
 - Уважайте согласие пользователей на обмен данными
 - Анonimizируйте/pseudonymizируйте данные где возможно
6. **Наблюдаемость**:
 - Журналируйте вызовы интеграций (успех/неудача, латентность)
 - Отслеживайте здоровье внешних сервисов
 - Оповещайте о сбоях интеграций

## Документация Интеграции
Каждая интеграция должна иметь:
- **API Контракт**: OpenAPI/Swagger или эквивалент
- **Метод Аутентификации**: OAuth, API key, и т.д.
- **Сопоставление Данных**: Как наши внутренние модели сопоставляются с внешними полями
- **Обработка Ошибок**: Конкретные коды ошибок и поведение резервирования
- **Рассмотрения Безопасности**: Шифрование, хранение токенов, и т.д.
- **План Тестирования**: Как протестировать интеграцию (mock сервис, песочница)
- **Рассмотрения Развертывания**: Переменные окружения, секреты, необходимые
