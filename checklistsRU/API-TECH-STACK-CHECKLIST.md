## 🔧 Решения по техническому стеку для документирования
- [x] docs/04-decisions/0001-tech-stack.md
  - Выбор библиотеки RBAC (контроль доступа на основе ролей):
    - Оценено CASL, AccessControl и собственное решение
    - Выбрана CASL с обоснованием (мигрируемость в MongoDB, выразительный синтаксис)
    - Включены примеры паттернов реализации (NestJS Guards, React hooks/HOCs)
  
  - Подход к документированию API:
    - Выбрана версия OpenAPI/Swagger 3.0.3
    - Инструменты для генерации/валидации: Swagger UI, Redoc, StopLight
    - Стратегия поддержания документации в синхроне: валидация в CI, pre-commit хуки
  
  - Рассмотрения шлюза/управления API:
    - Стратегия ограничения скорости: алгоритм token bucket с backend Redis
    - Централизация аутентификации/авторизации: валидация JWT на шлюзе
    - Требования к логированию и мониторингу: структурированные JSON-журналы, метрики Prometheus, correlation IDs
  
  - Подход к валидации данных:
    - Выбор библиотеки: Zod для frontend, class-validator для backend (NestJS)
    - Расположение слоя валидации: level контроллера DTOs с валидацией бизнес-правил на уровне сервиса
    - Стратегия форматирования ошибок: стандартный формат ошибок с кодом, сообщением, деталями
  
  - Стратегия версионирования API:
    - Выбрано версионирование в URI (/api/v1/) для ясности и управления кешированием
    - Политика устаревания: уведомление об устаревании за 6 месяцев, закат через 12 месяцев
    - Гарантии обратной совместимости: минорные версии обратно совместимы, мажорные версии могут вносить разрушительные изменения
