# Problem Statement: ZooLink

## 1. Background (Предпосылки)
Рынок услуг для владельцев домашних и сельскохозяйственных животных крайне фрагментирован. Основные каналы – доски объявлений типа Авито (отсутствие верификации, высокий риск мошенничества), закрытые группы в соцсетях (сложность поиска, нет репутационной системы) и специализированные форумы породных клубов (низкая доступность для новичков, географические ограничения). Владельцы питомников и фермеров тратят значительное время на ручной поиск пар для вязки, проверку документов и организацию встреч, while в то же время покупатели сталкиваются с отсутствием доверия к продавцам из-за невозможности проверить lineage животного, условия содержания или историю заболеваний. Существующие решения не обеспечивают необходимого уровня безопасности, прозрачности и удобства для масштабирования доверительных взаимодействий в этом сегменте.

## 2. Problem (Проблема)
**Владельцам питомников и фермеров сложно находить проверенных партнеров для вязки и крупных покупателей для племенного скота, тратя неделю+ на переговоры и верификацию документов через небезопасные каналы (телефон, мессенджеры).**  
**Частным лицам, продающим котят/щенят, неизвестно, как безопасно проверить намерения покупателя и оформить сделку без риска неоплаты или отказа после передачи животного.**  
**Покупателям сложно подтвердить здоровье, lineage и условия содержания животного перед покупкой, приводя к высокому проценту возвратов и споров после сделки.**  
**Все стороны сталкиваются с отсутствием единого репутационного профиля, который бы накапливал историю успешных сделок и снижал информационную ассиметрию.**  
*Примечание: Проблема сформулирована от лица стейкхолдеров без упоминания какой-либо платформы или технического решения.*

## 3. Business Objectives (Бизнес-цели по фазам)
**Фаза 1 (MVP – 3 месяца после запуска):**  
- Достичь 500+ объявлений о продаже/вязке домашних животных и 50+ объявлений о сельхозживотных без затрат на привлечение пользователей (рост через organic/партнерства).  
- Убедиться, что 70% опубликованных объявлений проходят предмодерацию с первого раза (качество контента).  
- Достичь удержания пользователей на 7-й день ≥20% (измеряется как повторный вход после регистрации).  

**Фаза 2 (Рост – 6 месяцев):**  
- Выйти на MRR 150 000₽ через мягкую монетацию (Boost-листингов, премиум-профили питомников, лидогенерация для ветклиник).  
- Обеспечить среднее время от объявления до сделки ≤14 дней для домашних животных и ≤30 дней для сельхоз (снижение цикла переговоров).  
- Достичь NPS ≥40 среди активных пользователей (измеряется через квартальные опросы).  

**Фаза 3 (Зрелость – 12 месяцев+):**  
- Закрыть 500+ сделок с комиссией в эскроу (только для сегмента livestock и дорогих пород pets, средний чек ≥50 000₽).  
- Подписать 3+ B2B контракта на SaaS-услуги для ферм/KХ (учет стада, аналитика).  
- Обеспечить 99.9% uptime для core API при пиковой нагрузке 1000 RPM.

## 5. Boundaries & Out of Scope (Границы системы)
**Что ТОЧНО не делаем на MVP:**  
- Не делаем реального времени чат (вместо этого – показ контактов продавца после модерации объявления).  
- Не делаем видеозвонки, трансляции или голосовые сообщения (это улучшает trust, но относится к Фазе 2+).  
- Не делаем мультиязычность (интерфейс и контент только на русском языке).  
- Не делаем генеалогическое древо с экспортом PDF (можно реализовать позже как премиум-фичу).  
- Не делаем форум или ленту новостей (социальный слой откладывается, фокус – на marketplace).  
- Не делаем интеграцию с Меркурием/ВетИС (регуляторные требования касаются только будущих фаз для livestock).  
- Не делаем биллинг или обработку платежей (все взаимодействия на MVP – бесконтактные; монетизация только неявная через партнерские ссылки и лиды для ветклиник).  

## 6. Key Constraints (Ключевые ограничения)
**Технические:**  
- Стек: Node.js (NestJS) + PostgreSQL (обусловлен доступностью команды и зрелостью инструментов для DDD).  
- Гео-поиск должен работать в радиусе от 1 до 100 км с откликом <1с на 95% запросов (требует индексации и кэширования).  
- Хранение фото – только через внешнее хранилище (S3-совместимое), локальное хранение ограничено 24h для модерации.  

**Юридические/Compliance:**  
- Предмодерация обязательна для всех объявлений (запрет на размещение контента, нарушающего ФЗ-152 о ПДн и ветеринарные правила).  
- Запрещено хранение паспортных данных пользователей без явного согласия (авторизация через телефон/OAuth минимизирует сбор ПДн).  
- Возрастной ценз 16+ (требует проверки при регистрации через согласие с условиями).  

**Операционные:**  
- Нет бюджета на 2+ нативных мобильных приложения на этапе MVP (выбор PWA как компромисс между охватом и стоимостью).  
- Ограничение команды: 1 бекенд-разработчик, 1 фронтенд-разработчик, 1 QA (поэтому архитектура должна исключать избыточную сложность).  
- Запрет на использование сторонних API с платными тарифами на этапе валидации гипотез (например, карты – только бесплатные tier Яндекс.Карт или Mapbox до лимита).  

**Бизнесовые:**  
- Запрет на любую прямую монетацию с пользователей на MVP (нельзя требовать оплаты за просмотр контактов, создание объявления или boosting) – это убивает network effect.  
- Обязательное разделение витрин для pets и livestock с разных точек входа (разные правила валидации полей, комиссии, UI-потоки).
