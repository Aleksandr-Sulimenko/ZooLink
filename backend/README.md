# ZooLink Backend (NestJS)

Modular monolith for the ZooLink MVP. The **canonical DB schema lives at `../database_schema.sql`**
(+ `../migrations/*.sql`); Prisma here is a generated **mirror** (ADR-0007), never authored by hand.

## Prerequisites
- Node 20 LTS
- PostgreSQL 14/16 reachable on `DATABASE_URL` (local dev: host PG14)
- Redis 7 reachable on `REDIS_URL`

## Local setup (no Docker)
```bash
# 1. DB role + database (once)
sudo -u postgres psql -c "CREATE ROLE zoolink LOGIN PASSWORD 'zoolink';" \
                      -c "CREATE DATABASE zoolink OWNER zoolink;" \
                      -c "ALTER ROLE zoolink CREATEDB;"

# 2. Load the canonical schema (31 tables)
PGPASSWORD=zoolink psql -h localhost -U zoolink -d zoolink -v ON_ERROR_STOP=1 -f ../database_schema.sql

# 3. Install deps + generate the typed Prisma client from the live DB
npm install
npm run db:sync          # = prisma db pull && prisma generate

# 4. Run
npm run start:dev        # API on http://localhost:3000  (Swagger: /docs)
npm run worker:dev       # background worker
```
`.env` (gitignored) holds local dev values; `../.env.example` + `../docker-compose.yml` are the canonical
topology for the full stack.

## Database workflow (SQL-canonical + Prisma introspect — ADR-0007)
The schema is **never** generated by Prisma Migrate. To change it:
1. Edit `../database_schema.sql` and add an idempotent `../migrations/YYYYMMDD_NNNN_*.sql`.
2. Update `../ZooLink_ERD.mmd`, `../docs/03-architecture/data-model.md`, and the counters in `../CLAUDE.md`.
3. Run it on a throwaway PG (twice — proves idempotency) + invariant negative tests.
4. `npm run db:sync` to refresh `prisma/schema.prisma` and the client.

Data access: **Prisma** for CRUD; **Kysely / parameterized `$queryRaw`** for geo (Haversine),
recursive pedigree (`WITH RECURSIVE`), and complex JSONB. Raw string interpolation is forbidden (ESLint-enforced).

## Endpoints (Phase 0)
- `GET /health/live` — liveness (no deps)
- `GET /health/ready` — readiness (PostgreSQL + Redis)
- `GET /metrics` — Prometheus
- `GET /docs` — Swagger UI (non-production)
- Domain routes are URI-versioned under `/v1/*`.

## Platform conventions
RFC 7807 errors (`application/problem+json`), `page`/`limit` pagination, `Idempotency-Key` (24h),
`ETag`/`If-Match` optimistic concurrency, Redis-backed rate limiting — all per
`../docs/03-architecture/api-contracts/API_CONVENTIONS.md`.
